# 项目章程：Awesome Humanoid Robot 人形机器人

> **版本**：草案  
> **状态**：v0.1.0 之前私有  
> **用途**：定义本项目是什么、为什么存在、如何组织、如何维护。

---

## 1. 使命

构建最全面、结构化、可验证的知识库，以回答以下问题：

> **如何实现人形机器人从0到1的量产与产业化应用？**

我们相信，人形机器人不仅仅是一个AI研究问题。它是一个系统问题，连接材料科学、精密制造、全球供应链、软件工程和真实世界部署。

这意味着追踪将一个想法变成可量产、可部署、经济上可行的人形机器人产品所需的每一个层次：原材料、零部件、制造工艺、设计工程、软件、AI、数据基础设施、组装、测试、量产、应用、市场、政策与监管。

我们不是在做算法参考书目。我们在绘制完整旅程的地图。

---

## 2. 愿景

一个活的、由社区共同维护的地图，让任何人都能回答以下问题：

- 需要哪些原材料和零部件？它们来自哪里？成本多少？
- 哪些制造工艺和供应链决策决定了一个设计能否规模化？
- 机械设计、AI、软件和数据基础设施在生产系统中如何交互？
- 从原型机到量产需要跨越什么？
- 哪些应用今天在经济上可行，哪些还需要技术或成本突破？
- 哪些政策、安全和监管框架会促进或阻碍部署？
- 在整个旅程中，瓶颈、风险和投资机会在哪里？

---

## 3. 核心原则

### 3.1 全链覆盖

我们不把人形机器人简化为AI模型。从矿山到市场的价值链每一层都在范围内。

### 3.2 可验证性

每一个论断都应可追溯至来源。不确定或未经验证的论断被明确标记。

### 3.3 结构化组织

扁平列表有用但不足。我们以**本体**形式组织知识——一张带关系的概念地图，而非只有类别。

### 3.4 AI辅助，人工复核

我们使用AI加速收集、摘要和综合。但所有公开论断在发布前都经过人工审核。

### 3.5 v0.1.0 后开放

本项目将在 v0.1.0 开源，以促进社区共创。在此之前保持私有，让核心团队建立坚实基础。

---

## 4. 范围边界

### 4.1 在范围内

- 机器人用原材料和关键矿产
- 零部件供应链（执行器、传感器、电池、计算单元、结构件）
- 制造工艺和质量控制
- 组装、集成和测试流程
- 量产、工厂设计和单位经济性
- 人形形态、运动学和机械设计
- AI模型：VLA、世界模型、locomotion/manipulation策略、规划、sim-to-real
- 软件栈：ROS 2、实时系统、仿真、数字孪生
- 数据集、基准、评估协议
- 公司、初创企业、研究机构和产业生态
- 应用和市场分析
- 法规、标准、安全和伦理

### 4.2 暂时不在范围内

- 非人形机器人（如工业机械臂、无人机、四足机器人），除非直接与人形供应链相关
- 未应用于机器人领域的一般AI研究
- 缺乏扎实分析的科幻或长期预测
- 投资建议

## 5. 产业链本体

本项目围绕以下十个领域组织。每个领域在 [`docs/ontology/`](ontology/) 中有独立文档。

```
上游
├── 01 原材料与关键资源
├── 02 零部件与子系统
└── 03 制造工艺

中游
├── 04 组装、集成与测试
├── 05 量产与规模化
└── 06 设计与工程

智能层
├── 07 AI模型与算法
├── 08 软件与中间件
└── 09 数据与数据集

验证与部署
├── 10 评测与基准
├── 11 应用与市场
└── 12 政策、监管与伦理
```

本体不是平等类别的分类法。它是一张**价值链地图**：上游变化（如稀土价格）会向下游传导（如执行器成本、机器人BOM、市场价格）。

## 6. AI4Sci 工作流

### 6.1 输入

- 学术论文（arXiv、会议、期刊）
- 公司新闻稿、投资者报告和拆解分析
- 专利申请
- 行业报告和供应链分析
- 政府政策文件和标准
- 技术博客和访谈

### 6.2 提取模板

对每个条目，我们记录：

| 字段 | 说明 |
|------|------|
| **名称** | 公司、论文、零部件或概念 |
| **领域** | 在产业链本体中的位置 |
| **来源** | 原始URL或引用 |
| **状态** | 已验证 / 部分验证 / 未验证 / 推测 |
| **摘要** | 一段话描述 |
| **相关性** | 对人形机器人的重要性 |
| **交叉链接** | 其他领域的相关条目 |
| **最后复核** | 人工或AI复核日期 |

### 6.3 验证等级

| 等级 | 含义 |
|------|------|
| ✅ 已验证 | 有原始来源支持，并已复核 |
| ⚠️ 部分验证 | 有二手来源支持，或基于合理推断 |
| ❓ 未验证 | 来源中提及，但未独立确认 |
| 🔮 推测 | 前瞻性或分析性论断，明确标注 |

## 7. 治理与发布计划

### 7.1 v0.1.0 之前（私有）

- 核心团队整理本体和初始内容
- AI辅助文献扫描和摘要
- 所有条目在 v0.1.0 之前内部复核

### 7.2 v0.1.0（公开）

- 仓库转为公开
- 发布贡献指南
- 本体覆盖全部十个领域，每个领域至少有一条经过整理的条目

### 7.3 v0.1.0 之后

- 通过 pull request 接受社区贡献
- 季度发布周期，附 changelog
- 每个本体分支设有领域维护者

---

## 8. 成功指标

- 本体覆盖全部十个领域，并明确表达关系。
- v0.1.0 时每个领域至少有一条已验证条目。
- 每一条公开论断都可追溯至来源。
- v0.1.0 之后有活跃的社区贡献。
