# Stage 0 기초 기반 다지기: 휴머노이드 로봇 제작의 입장권 획득

> ⚠️ 본 로드맵 내용은 공개 자료와 이론적 지식을 바탕으로 정리되었으며, 실제 기기 검증을 거치지 않았습니다. 전기 및 기계 작업 시 안전 규정을 직접 확인하시기 바랍니다.

## 본 단계의 위치

Stage 0의 목표는 "모든 기초를 다 배우는 것"이 아닙니다. 그것은 끝없는 구렁텅이이며, 수많은 사람들이 그 안에서 좌절합니다. 목표는 네 가지 입장권을 획득하는 것입니다:

1. **수학**: 제어 논문에서 나오는 행렬, 확률, 동역학 방정식, QP가 낯설지 않아야 합니다;
2. **프로그래밍 및 도구 체인**: Python/C++/git/ROS 2를 실무에 사용할 수 있고, 3D 프린팅이 무엇인지 알아야 합니다;
3. **회로 개념**: 전기가 어디서 오는지, 신호가 어떻게 흐르는지 알아야 하며, 이후 실제 하드웨어를 다룰 때 보드를 태우지 않아야 합니다;
4. **시뮬레이션**: URDF 파일 하나를 읽고, MuJoCo에서 이족 모델을 일으켜 세울 수 있어야 합니다.

- **시간 예산**:业余 6–10주 (주당 8–10시간), 전일제 3–4주.
- **예산**: 약 ¥0 – 본 단계 소프트웨어는 모두 오픈소스입니다. 선택적 3D 프린팅 시험 제작 비용은 공급업체에 직접 확인해야 합니다.
- **핵심 규율**: 각 주제에 대해 "어느 정도까지 배우면 충분한지"에 대한 판단 기준을 제시하며, **기준에 도달하면 즉시 넘어가고, 뒤돌아 문제를 반복해서 풀지 마십시오**. 실제 부족한 점은 이후 단계에서 드러나며, 그때 구체적인 문제를 가지고 돌아와 보충하는 것이 효율이 10배 높습니다.

## 단계 1: 수학 네 가지

수학과처럼 (정의-정리-증명) 방식으로 배우지 말고, "제어 엔지니어의 최소 무기고" 방식으로 배우십시오. 네 과목의 판단 기준은 모두 **시연 가능**해야 하며, "내가 이해한 것 같다"가 아닙니다.

### 1.1 선형대수학 (Linear Algebra)

【할 일】벡터 및 행렬 연산, 고유값/고유벡터, 회전 행렬과 동차 변환, 최소제곱법과 의사역행렬. 실습: NumPy로 4×4 동차 변환을 작성하여 발끝 좌표를 발목 관절 좌표계에서 몸통 좌표계로 변환하고, 2링크 평면 팔의 정기구학을 작성하십시오.

【이유】선형대수학은 휴머노이드 로봇의 모든 공간 관계를 표현하는 언어입니다: 정기구학은 동차 변환의 연속 곱셈이며, 야코비 행렬은 관절 속도를 말단 속도로 매핑하고, 전신 제어의 모든 QP는 행렬 연산입니다. 카드: [선형대수학](/entry/ent_foundation_linear_algebra/) (벡터 공간, 선형 변환, 행렬 및 선형 방정식 시스템을 연구하는 수학 분야). Wiki [제8장](/wiki/chapters/chapter-08/)의 기구학 유도는 모두 이 위에 구축됩니다.

【상황별 분석】CS 배경: 계산은 이미 익숙하므로, **기하학적 직관**에 집중하십시오 – 회전 행렬은 아홉 개의 숫자가 아니라, 하나의 좌표계의 세 축입니다; 물리/수학 배경: 위 실습을 바로 수행하여 검증하십시오; 초보자: 먼저 3Blue1Brown의 《선형대수학의 본질》을 보고 직관을 쌓은 후, 직접 계산해 보십시오.

**어느 정도까지 배우면 충분한가** (세 가지 모두 통과 시 중단): ① 2D 회전 행렬을 직접 유도하고 각 열의 기하학적 의미를 설명할 수 있어야 함; ② 2링크 평면 팔 정기구학 코드가 실행되고, 출력이 수동 계산과 일치해야 함; ③ 야코비의 의사역행렬 J⁺를 보고 최소제곱 문제를 푸는 것임을 알 수 있어야 함.

### 1.2 확률론 (Probability Theory)

【할 일】확률 변수, 기댓값과 공분산, 가우스 분포, 베이즈 정리, 조건부 및 주변화. 실습: NumPy로 가우스 잡음이 있는 "관절 엔코더 판독값"을 시뮬레이션하고, 히스토그램을 그리고 평균/분산을 계산하십시오; 또한 베이즈 정리를 사용하여 "센서 경보 → 실제 고장 확률" 문제를 수동으로 계산하십시오.

【이유】실제 센서는 항상 잡음을 포함하며, 상태 추정 (칼만 필터 계열)은 본질적으로 순환적 베이즈 추론입니다; 이후 사용될 모방 학습, VLA 모델도 결정론적 값이 아닌 동작의 확률 분포를 출력합니다. 카드: [확률론](/entry/ent_foundation_probability_theory/) – 로봇 공학 및 머신러닝의 모든 확률 모델의 기초.

【상황별 분석】AI 배경: 판단 기준으로 자가 점검하면 대부분 바로 통과합니다; 하드웨어 배경: 이 과목이 네 과목 중 가장 약할 가능성이 높으며, 상태 추정을 이해하는 데 직접적인 영향을 미치므로 – 수학 예산의 절반을 여기에 투자할 가치가 있습니다; 초보자: 선형대수학 이후에 배우십시오.

**어느 정도까지 배우면 충분한가**: ① 공분산 행렬의 대각선 요소와 비대각선 요소가 각각 무엇인지 한 문장으로 설명할 수 있어야 함; ② 베이즈 문제를 수동으로 계산하고 수치가 정확해야 함; ③ 칼만 필터의 다섯 가지 공식을 보고 당황하지 않아야 함 – 유도를 요구하지는 않지만, 각 공식의 역할 (예측, 이득, 업데이트)을 설명할 수 있어야 함.

### 1.3 고전역학 (Classical Mechanics)

【할 일】뉴턴-오일러 방정식, 라그랑주 방법, 운동 에너지/위치 에너지, 관성 텐서, 각운동량. 실습: 라그랑주 방법을 사용하여 단진자, 이중진자의 동역학 방정식을 작성하고, 수치 적분으로 5초 동안 시뮬레이션하여 각도 곡선을 그리십시오.

【이유】휴머노이드 로봇은 부유 기저 다강체 시스템이며, M(q)q̈ + C(q,q̇) + g(q) = τ 방정식은 이후 읽게 될 모든 제어 논문에 등장합니다; 이족 보행 균형 (ZMP, 포착점)은 모두 고전역학의 직접적인 결과입니다. 카드: [고전역학](/entry/ent_foundation_classical_mechanics/) (뉴턴 법칙, 보존 원리 및 강체 동역학 포함). Wiki [제8장](/wiki/chapters/chapter-08/)과 [제14장 로봇 제어 기초](/wiki/chapters/chapter-14/)의 유도는 모두 여기서 시작됩니다.

【상황별 분석】기계/자동차 배경: 이 과목은 대부분 충분하므로, 이중진자 실습으로 검증하고 넘어가십시오; 순수 소프트웨어 배경: 이 단계에서 **가장 중요한** 보충 과목입니다 – 역학적 직관이 없으면, 시뮬레이션에서 로봇이 넘어질 때 제어기의 잘못인지 모델의 잘못인지 구분할 수 없습니다.

**어느 정도까지 배우면 충분한가**: ① 이중진자 방정식을 유도할 수 있어야 함 (책을 보면서 유도하는 것은 허용되지만, 책을 덮고 각 단계가 무엇을 하는지 설명할 수 있어야 함); ② "왜 몸통 관성이 큰 휴머노이드 로봇이 오히려 더 안정적으로 서 있을 수 있는지"를 정성적으로 설명할 수 있어야 함; ③ URDF 파일의 `<inertial>` 태그 각 필드의 의미를 이해할 수 있어야 함 – 이는 단계 4를 위한 직접적인 준비입니다.

### 1.4 볼록 최적화 (Convex Optimization)

【할 일】볼록 집합/볼록 함수 판별, 표준 QP (2차 계획법) 형식, 등식/부등식 제약 조건의 기하학적 의미, 기성 솔버를 호출하여 QP를 푸는 것. 실습: Python으로 min ‖x‖² s.t. Ax=b, x≥0 문제를 푸십시오 (솔버는 OSQP, quadprog 등 환경에 맞게 선택).

【이유】휴머노이드 제어의 실시간 계층은 거의 모두 QP를 풉니다: 전신 제어 (WBC)는 "작업 추적 + 접촉력 제약 + 토크 상한"을 역동역학 QP로 작성하여 매 밀리초마다 풉니다; MPC는 순환적으로 QP를 풉니다. 볼록성은 모든 지역 최적해가 전역 최적해임을 보장합니다 – 이것이 엔지니어가 QP를 kHz급 제어 루프에 넣을 수 있는 근본적인 이유입니다. 카드: [볼록 최적화](/entry/ent_foundation_convex_optimization/). 자세한 내용은 Wiki [제14장](/wiki/chapters/chapter-14/)을 참조하십시오.

【상황별 분석】이 과목은 네 가지 중 **한계 효용 체감이 가장 빠른** 과목입니다: 직접 제어기를 작성하지 않는다면, "볼록성 판별 + 솔버 호출 + QP 구조 이해"만으로 충분하며, KKT 유도는 필요할 때까지 미루십시오. OR 배경은 자가 점검으로 통과합니다.

**어느 정도까지 배우면 충분한가**: ① 주어진 문제가 볼록한지 판별할 수 있어야 함 (2차 양정의 목적 함수 + 선형 제약 조건 → 볼록함); ② 연습 문제의 해가 모든 제약 조건을 만족하는지 확인할 수 있어야 함; ③ 다른 사람에게 "왜 WBC가 실시간 루프에서 QP를 사용할 수 있는지" 설명할 수 있어야 함.

## 단계 2: 프로그래밍 및 도구 체인

【할 일】① Python 능숙도: NumPy 벡터화, matplotlib 그래프 그리기, class 작성 가능; ② C++ "읽고 수정 가능": 포인터/참조, 헤더 파일, 컴파일/링크; ③ git 일상 작업: clone, branch, commit, push, PR 제출; ④ ROS 2 LTS 설치, talker/listener 실행 및 자체 발행/구독 노드 쌍 작성; ⑤ FDM 3D 프린팅 기본 프로세스 이해 (모델링 → 슬라이싱 → 프린팅) 및 강도 상식.

【이유】휴머노이드 소프트웨어 스택의 사실상 표준은 [ROS 2 미들웨어](/entry/ent_software_ros_2_middleware_2024/)입니다 – DDS 기반 발행/구독 메커니즘, 실시간 지원 (출처: ros.org, 카드 참조). 이후 단계의 시뮬레이션, 구동, 상태 추정은 모두 노드/topic으로 구성됩니다. 하위 레벨 실시간 루프는 C++, 알고리즘 프로토타입은 Python을 사용하는 것이 업계의 기본 분업입니다. 3D 프린팅은 이후 전체 기계 단계의 주요 제조 수단입니다: 오픈소스 휴머노이드 Berkeley Humanoid Lite의 구조 부품과 사이클로이드 감속기는 모두 프린팅 부품이며, 전체 기계 BOM은 미국 조달 시 약 $4,312, 중국 조달 시 약 $3,236입니다 (출처: `data/roadmap/research/berkeley-humanoid-lite.md`) – 지금 배우는 슬라이싱 및 공차 상식은 그때 직접 비용과 시간으로 이어집니다. 소프트웨어 생태계 전망은 Wiki [제22장 소프트웨어 미들웨어](/wiki/chapters/chapter-22/)를 참조하십시오.

【상황별 분석】순수 소프트웨어 배경: Python/git 자가 점검 통과 시 건너뛰고, C++ 메모리 및 실시간 개념, ROS 2의 노드/topic/service 세 가지에 시간을 투자하십시오; 하드웨어 배경: NumPy로 Python을 시작하는 것이 가장 효율적이며, MATLAB 사고 방식에 가장 가깝습니다; 초보자: Python을 먼저 배우고 git은 그 다음, C++은 필요할 때까지 미루십시오 – 배워도 사용하지 않으면 잊어버립니다.

**어느 정도까지 배우면 충분한가**: ① ROS 2: 두 개의 노드를 독립적으로 작성하여 하나는 발행하고 하나는 구독하며, `ros2 topic echo`로 데이터 흐름을 확인할 수 있어야 함; ② git: fork→branch→commit→PR 전체 프로세스를 한 번 독립적으로 완료 (자신의 연습 저장소로 연습); ③ 3D 프린팅: FDM 레이어 방향이 부품 강도에 미치는 영향과, 왜 맞춤 구멍에 공차를 남겨야 하는지 설명할 수 있어야 함 (구체적인 기계의 공차 수준은 공급업체에 직접 확인 필요).

## 3단계: 회로 및 임베디드 기초

**【할 일】** 이 단계에서는 보드를 제작하지 않고, 세 가지 개념을 확립합니다: **전원 공급**(전압 레일, 전류 예산, 선경과 전압 강하, 퓨즈와 스위치), **접지**(공통 접지, 접지 루프), **신호**(디지털 레벨, 차동 전송, 노이즈 내성). 그런 다음 [CAN 버스](/entry/ent_technology_can_bus_2024/)의 역할에 중점을 둡니다: 휴머노이드 로봇 관절 모터 드라이버와 중앙 컨트롤러를 연결하는 필드버스입니다(출처: 카드, Wikipedia CAN bus 2024). 선택적 실습: 입문용 CAN 어댑터를 하나 구매하여(가격은 공급업체에 직접 확인 필요), can-utils를 사용해 실제 버스 데이터를 캡처하여 프레임 구조를 확인합니다.

**【이유】** 전체 시스템 단계에서는 수십 개의 관절 드라이버가 버스에 연결됩니다: Berkeley Humanoid Lite는 CAN을 사용하여 각 액추에이터를 연결하며, 메인 컨트롤러는 Intel N95 미니 PC(약 $129, 동체에 장착, 저수준 제어 및 RL 정책 동시 실행, 출처: `data/roadmap/research/berkeley-humanoid-lite.md`)입니다. 전체 시스템 디버깅 시 절반 이상의 "미스터리한 문제"는 전기적 계층(전압 강하, 공통 접지, 종단 저항, 배선)에서 발생합니다. 지금 개념을 확립하면 나중에 보드가 타는 것을 줄일 수 있습니다. 전원 시스템 공학 세부 사항은 Wiki [제6장 계산, 전원 및 열 관리](/wiki/chapters/chapter-06/)를 참조하십시오.

**【상황별 분석】** 전기/자동화 배경: 이 섹션은 30분 안에 개념을 훑어보면 됩니다. 소프트웨어 및 비전공자: 회로 설계를 할 수 있을 필요는 없지만, "측정할 용기와 능력"을 갖추어야 합니다. 즉, 멀티미터로 전압과 단선/단락을 측정하고, 24V 버스 단락이 무엇을 의미하는지 알아야 합니다. 안전 주의: 리튬 배터리와 버스 커패시터는 장난감이 아닙니다. 전원이 인가된 상태에서 작업하기 전에 안전 규정을 직접 확인하고, 확신이 없으면 하드웨어 경험이 있는 사람과 함께하십시오.

**어느 정도면 충분한가**: ① "배터리 → 퓨즈/비상 정지 → 버스 → 각 드라이버"의 전원 공급 토폴로지 스케치를 손으로 그릴 수 있어야 합니다. ② CAN 버스 양 끝에 120Ω 종단 저항을 추가해야 하는 이유를 설명할 수 있어야 합니다. ③ 차동 신호가 단일 종단 신호보다 노이즈 내성이 뛰어난 이유를 설명할 수 있어야 합니다.

## 4단계: 시뮬레이션 첫 경험 – URDF 이해하고, 이족 보행 로봇을 일으켜 세우기

**【할 일】** ① [MuJoCo 물리 엔진](/entry/ent_software_mujoco_physics_engine_2022/)을 설치합니다. ② 오픈소스 휴머노이드/이족 보행 로봇의 [URDF 로봇 설명 형식](/entry/ent_technology_urdf_robot_description_format_2024/) 파일을 찾아 정독합니다. URDF는 XML을 사용하여 로봇의 링크, 관절, 관성 및 형상을 설명하는 형식입니다(출처: wiki.ros.org/urdf, 카드 참조). ③ 모델을 MuJoCo에 로드합니다(또는 프로젝트 자체 MJCF 사용). ④ 가장 기본적인 위치 PD 컨트롤러를 작성합니다: 서 있는 자세의 관절 목표 각도를 지정하여 이족 보행 모델이 10초 동안 제자리에 서 있도록 합니다.

**【이유】** MuJoCo는 풍부한 접촉 역학을 갖춘 고충실도 물리 엔진으로, 휴머노이드 제어 연구에 널리 사용됩니다(출처: mujoco.org, 카드 참조). 이족 보행의 핵심 난제는 발바닥 접촉과 균형이며, 이것이 MuJoCo의 강점입니다. URDF를 읽는 것은 오픈소스 휴머노이드 저장소를 여는 첫 번째 열쇠입니다: 실제 오픈소스 프로젝트는 URDF/MJCF/USD 세 가지 설명 형식을 동시에 유지 관리합니다(Berkeley Humanoid Lite, 출처: `data/roadmap/research/berkeley-humanoid-lite.md`). 이 형식을 이해하면 다른 사람의 로봇 "뼈대 도면"을 읽을 수 있습니다. 시뮬레이션 기술 스택에 대한 체계적인 논의는 Wiki [제23장 시뮬레이션 및 물리 엔진](/wiki/chapters/chapter-23/)을 참조하십시오.

**【상황별 분석】** GPU가 없어도 괜찮습니다. 서 있는 수준의 강체 시뮬레이션은 CPU로 충분합니다. 모델 선택 시 "크기가 작고 문서가 완전한" 오픈소스 이족 보행 로봇을 우선시하십시오(조사 파일 `data/roadmap/research/`에는 Berkeley Humanoid Lite, ToddlerBot 등의 후보와 비용 및 난이도 분석이 있습니다). 처음부터 전체 크기 모델을 시도하지 마십시오. **이 단계에서 한 가지만 완료할 수 있다면, 이것을 완료하십시오.** 역학, 프로그래밍 및 도구 체인 세 가지를 동시에 검증합니다.

**어느 정도면 충분한가**: 아래 검증 기준 1, 2항목입니다. 이 단계가 Stage 0의 졸업 설계입니다.

## 검증 기준

항목별로 자체 점검하고, 모두 통과해야만 Stage 1로 진행할 수 있습니다:

1. **URDF 읽기 이해**: 본 적 없는 이족 보행 URDF를 보고, 문서 없이 구두로 답변할 수 있어야 합니다. 제어 가능한 관절 수, 루트 링크 위치, 발목 관절의 `origin`/`axis`, `<inertial>`의 `ixx/iyy/izz` 의미. 또한 `check_urdf` 또는 동등한 도구를 사용하여 구문 분석 오류가 없음을 확인합니다.
2. **MuJoCo 서 있기**: 이족 보행 모델을 로드하고, 직접 작성한 위치 PD 제어를 사용하여 외력 없이 10초 이상 서 있어야 합니다. 몸통에 작은 충격을 가한 후 모델이 다시 서 있을 수 있거나, 그렇지 못한 이유(게인, 접촉 또는 모델 문제)를 올바르게 설명할 수 있어야 합니다. 이 항목은 역학적 직관을 테스트하는 것이지, 파라미터 튜닝 운을 테스트하는 것이 아닙니다.
3. **수학 확인**: 네 가지 세트의 "충분 조건"에서 각각 무작위로 하나를 뽑아, 즉석에서 시연(수동 계산 또는 코드 실행)할 수 있어야 합니다.
4. **도구 체인**: ROS 2 talker/listener 실행 기록(터미널 스크린샷 또는 녹화)이 있어야 합니다. 이 단계의 모든 연습 코드는 git 저장소에 있으며, 커밋 기록이 명확하게 읽혀야 합니다.
5. **회로 개념**: 동료(또는 허공)에게 전원 공급 토폴로지, CAN 종단 저항 및 차동 신호 세 가지를 설명할 수 있어야 합니다. 설명하지 못하면 미달로 간주합니다.

두 가지 "충분하지 않음" 경고 신호: 튜토리얼 없이 PD 컨트롤러를 작성할 수 없는 경우(4단계는 따라하기임). URDF의 `origin`을 변경했을 때 모델이 왜 떠다니는지 모르는 경우(1.1 기하학적 직관 미확립). 둘 중 하나라도 해당되면 해당 소절로 돌아가 재확인합니다.

## 일반적인 문제 및 해결

| 증상 | 가능한 원인 | 확인 조치 |
|------|----------|----------|
| MuJoCo에서 모델 로드 시 즉시 분해/날아감 | 초기 자세 관통(관절이 서로 겹침); 특정 링크에 `<inertial>`이 없어 질량/관성이 0임 | 시뮬레이터 컴파일 경고 확인; 모델에 합리적인 초기 키프레임 제공; 각 링크의 관성 매개변수 확인 |
| PD 서 있을 때 관절 고주파 떨림 | 게인 과다, 적분 단계 크기 과다, 수치적 강성 | 먼저 P를 절반으로 줄인 후 천천히 다시 증가; 시뮬레이션 timestep을 줄여 재현 및 비교; 기대 감쇠를 스프링 강성으로 잘못 작성했는지 확인 |
| 발바닥이 미끄러져 빙판 위를 걷는 듯함 | 접촉 마찰 매개변수가 기본값을 사용하여 실제 밑창-지면 조합보다 낮음 | geom/접촉 쌍의 friction 매개변수 확인; 밑창 재료(고무/인쇄 PLA)에 따라 마찰 계수 크기 조정, 구체적인 값은 재료에 따라 직접 확인 필요 |
| URDF를 MJCF로 변환 후 관절 운동 방향 반대 | URDF `axis` 정의가 대상 시뮬레이터 규칙과 일치하지 않음; 각도 단위 혼용(도 vs 라디안) | 뷰어에서 단일 관절을 하나씩 구동하여 방향 확인; 전체 텍스트 검색으로 각도 필드 단위 확인 |
| ROS 2 두 노드가 서로 메시지를 수신하지 못함 | `ROS_DOMAIN_ID` 불일치; 방화벽이 DDS 트래픽 차단; 토픽 이름/QoS 불일치 | 먼저 동일 머신에서 실행 후 크로스 머신 시도; `ros2 topic list`로 양쪽이 동일한 토픽을 보는지 확인; 환경 변수 확인(사용 중인 배포판 공식 문서 기준) |
| Stage 1 진입을 계속 망설이며 수학을 반복 학습 | 완벽주의 함정: "기초 다지기"를 지연 핑계로 사용 | 검증 기준을 항목별로 체크하고 통과하면 바로 진행; 실제 부족한 점은 Stage 1에서 드러나게 함 |

이상으로 Stage 0이 완료되었습니다. 다음은 [Stage 1 관절(액추에이터) 만들기](stage-1-actuator.md)입니다: 3단계의 회로 개념과 PD 제어를 실제 하드웨어에 처음 적용합니다. URDF와 MuJoCo의 기초는 [Stage 2 이족 보행 플랫폼](stage-2-biped.md)의 시뮬레이션 보행에서 활용됩니다.
