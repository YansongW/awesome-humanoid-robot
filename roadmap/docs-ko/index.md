# 0→1 휴머노이드 로봇 제작: 로드맵 개요

> ⚠️ 본 로드맵의 내용은 공개 자료와 이론적 지식을 바탕으로 정리되었으며, 실제 기기 검증을 거치지 않았습니다. 전기 및 기계 작업 시 안전 규정을 직접 확인하시기 바랍니다.

!!! warning "이것부터 읽고, 로봇 제작을 논하세요"
    이 로드맵은 **실제 하드웨어**를 다루는 과정입니다: 리튬 배터리, 24–48 V 버스, kg 단위로 움직이는 금속 구조물, 정지 시 손가락이 끼일 수 있는 관절 토크. 이 로드맵의 어떤 페이지도 안전 매뉴얼을 대체할 수 없습니다. 전원을 켜기 전에 전기 및 기계 안전 규정을 직접 확인하십시오; 확신이 없는 작업은 하드웨어 경험이 있는 사람과 함께 진행하세요. 모든 가격, 납기, 공급업체 정보는 주문 시점의 실제 상황을 기준으로 합니다.

## 이 저장소의 세 가지 용도: 검색, 학습, 제작

이 저장소는 동시에 "지도", "책", "시공도" 역할을 합니다. 동일한 개념이 세 계층에서 각각 한 번씩 등장하며 역할이 다릅니다:

| 용도 | 진입점 | 언제 사용하는가 | 예시 |
|------|--------|----------------|------|
| **검색**: 지식 그래프 | [/search/](/search/) 및 카드 페이지 | 생소한 용어를 만나거나 특정 개체의 매개변수/출처/상하위 관계가 필요할 때 | [CAN 버스](/entry/ent_technology_can_bus_2024/)가 무엇인지, 어디에 사용되는지 확인 |
| **학습**: Wiki 저서 | [/wiki/](/wiki/) | 특정 분야를 체계적으로 이해해야 할 때 (30장 + 부록, 연구 논문 기준으로 작성) | [제14장 로봇 제어 기초](/wiki/chapters/chapter-14/)를 읽어 WBC 이해 |
| **제작**: 본 로드맵 | 현재 보고 있는 이 계층 | 휴머노이드 로봇을 0에서 1까지 직접 만들고 싶을 때 | [0단계](stage-0-foundations.md)의 검증 기준을 항목별로 통과 |

세 계층의 일반적인 연동 방식: 로드맵에서 특정 단계를 진행하다가 → 막히는 개념이 있으면 카드 페이지에서 매개변수와 출처 확인 → 완전히 이해하고 싶으면 Wiki 해당 장에서 체계적으로 학습. 로드맵은 Wiki의 긴 설명을 반복하지 않고, "이 단계에서 무엇을 할 것인지, 왜 하는지, 제대로 되었는지 어떻게 판단할지"만 알려줍니다.

## 로드맵 페이지 읽는 법: 3단 구성과 데이터 규율

본 로드맵의 각 실무 단계는 3단 구성으로 되어 있으며, 이것이 일반 튜토리얼과의 차이점입니다:

1. **【무엇을 할 것인가】** 구체적이고 실행 가능하며 완료 표시가 있는 동작;
2. **【왜 하는가】** 이 단계의 배경 원리와 해당 지식 카드 및 Wiki 장으로의 링크, 더 깊이 파고들고 싶다면 진입점 제공;
3. **【당신의 상황에 따른 분석】** 자신의 예산, 시나리오, 기술 배경에 따라 선택할 수 있도록 안내 — 동일한 목표라도 순수 소프트웨어 배경과 기계 배경의 접근 방식이 다릅니다.

데이터 규율: 본문에 등장하는 구체적인 매개변수(가격, 토크, 자유도, 소비 전력 등)는 모두 가까운 위치에 출처를 표시합니다(개체 카드 또는 `data/roadmap/research/` 조사 파일); 신뢰할 수 있는 출처를 찾을 수 없는 경우 "공급업체에 직접 확인 필요"라고 명시하며, 숫자를 임의로 작성하지 않습니다.

## 4단계 지도

| 단계 | 목표 | 예상 시간 (취미 투자) | 예산 규모 | 검증 (판단 기준 요약) |
|------|------|---------------------|----------|------------------|
| **Stage 0 기초 다지기**[→ 페이지](stage-0-foundations.md) | 입장권 획득: 수학 4종 세트, Python/C++ 및 ROS 2, 회로 개념, URDF 읽기 | 6–10주 (주당 8–10시간) | ≈ ¥0: 소프트웨어 전부 오픈소스; 선택적으로 3D 프린팅 시험 가능, 비용은 공급업체에 직접 확인 | 이족 보행 URDF 하나 읽기; MuJoCo에서 이족 모델을 10초 이상 서 있게 하기 |
| **Stage 1 관절 만들기**[→ 페이지](stage-1-actuator.md) | 작업대에서 검증된 액추에이터 모듈 제작: 모터 + 감속기 + 센서 + 드라이버 + 제어 전체 경로 이해 | 4–8주 (추정) | 단일 관절 수백~수천 위안 수준: QDD 방식 단일 BOM 약 $157–188 (Berkeley 6512, 출처: `data/roadmap/research/berkeley-humanoid-lite.md`); 서보/하모닉 방식 가격은 공급업체에 직접 확인 | 성능 지표표 문서화; 30° 계단 응답 오버슈트 < 10%; 1 Hz 정현파 추적 진폭 감쇠 < 10%; 통신 링크 1시간 연속 무장애 |
| **Stage 2 이족 플랫폼**[→ 페이지](stage-2-biped.md) | 시뮬레이션에서 제어 스택을 먼저 실행한 후, 오픈소스 이족/바퀴형 플랫폼을 복제하여 평지 안정 보행 구현 | 3–6개월 (추정) | 플랫폼에 따라 예산 결정: Upkie 약 $3,000, Berkeley Humanoid Lite 약 $3,236–4,312, ToddlerBot 약 $6,000 (출처: `data/roadmap/research/` 해당 파일); 약 ¥2만–5만 수준, 환율 및 조달 채널 변동은 직접 계산 필요 | 시뮬레이션 연속 보행 10분 이상 넘어지지 않음; 실제 기기 5분 이상 서 있기, 평지 연속 보행 10분 이상, 가벼운 밀기 회복 가능, 비상 정지 시연 가능 |
| **Stage 3 완전한 휴머노이드**[→ 페이지](stage-3-humanoid.md) | 이족 보행 위에 상체 추가: 팔, 말단 실행기, 인식 스택 및 지능 계층, 종단 간 작업 완료 | 6개월 이상, 지속적 반복 | 추가 수천~수만 위안 (팔/손/카메라/연산 보드, 선택한 방식에 따라 직접 계산) | 음성 명령 듣기 → 자율적으로 테이블까지 이동 → 컵 인식 및 집기; IK 무작위 샘플링 해결 성공률 ≥ 95%, 말단 도달 오차 ≤ 2 cm |

두 가지 설명:

- **시간은 추정치**이며, 주당 8–10시간 취미 투자 기준; 전업으로 하면 약 1/3로 단축 가능. 배경 지식에 따라 크게 달라지므로 다음 절 참조.
- **예산에는** 이미 보유한 일반 장비(컴퓨터, 멀티미터, 인두)가 포함되지 않습니다. 상용 완제품 플랫폼은 다른 가격대입니다: ROBOTIS OP3 판매 가격 $13,764.35 (출처: `data/roadmap/research/robotis-op3-darwin-op.md`, 2026년 페이지 스냅샷), 이것이 본 로드맵 Stage 2에서 완제품 대신 오픈소스 플랫폼을 권장하는 이유 중 하나입니다.

## 이 로드맵의 경계

"무엇을 하지 않는지"를 명확히 하는 것이 "무엇을 한다"고 약속하는 것보다 여러분의 시간을 보호해 줍니다:

- **논문 소개 목록이 아닙니다**. 최신 알고리즘(VLA, 세계 모델 등)은 Wiki 18–20장에서 체계적으로 다루며, 로드맵은 "로봇 제작에 필요할 때"만 이를 가리킵니다. 먼저 로봇을 일으켜 세우고, 그다음에 똑똑하게 만드는 것을 논하세요.
- **처음부터 모든 것을 자체 개발하는 경로가 아닙니다**. Stage 2는 성숙한 오픈소스 플랫폼 위에서 첫 번째 로봇을 만들 것을 명확히 권장합니다: 20+ 자유도의 구조와 구동을 처음부터 설계하는 것은 팀 수준, 다중 기기 반복 수준의 작업이며, 개인 첫 번째 로봇의 합리적인 목표가 아닙니다. 조사 파일 `data/roadmap/research/`에는 Berkeley Humanoid Lite, ToddlerBot, Poppy, ROBOTIS OP3 등 후보 플랫폼에 대한 비용 및 진입 장벽 분석이 각각 포함되어 있으므로, 선택 시 직접 읽으십시오.
- **구매 조언이나 안전 매뉴얼이 아닙니다**. 모든 플랫폼 비교는 출처가 확인 가능한 사실과 trade-off만을 제시합니다; 주문 전에 최신 가격, 라이선스(일부 오픈소스 설계 파일은 비상업적 라이선스, 예: ToddlerBot, 출처: `data/roadmap/research/toddlerbot.md`) 및 규정 준수 요구 사항을 직접 확인하십시오.
- **직선 진행을 보장하지 않습니다**. 실제 상황에서는 Stage 1에서 드라이버 하나를 태워 Stage 0 3단계로 돌아가 전기 공부를 다시 하거나, Stage 2 시뮬레이션이 안 맞아 1단계로 돌아가 역학을 보충하는 경우가 자주 있습니다. 로드맵의 가치는 여러분이 현재 어디에 있고, 다음에 어디로 가야 하는지를 알려주는 것이지, 되돌아가는 것을 금지하는 것이 아닙니다.

## 배경에 맞는 진입점 선택

### 순수 소프트웨어 배경 (CS / AI / 알고리즘 엔지니어)

당신의 약점은 코드가 아니라 **물리적 직관과 전기적 상식**에 있습니다. 제안:

1. 수학 4종 세트 중 [고전 역학](/entry/ent_foundation_classical_mechanics/)만 보충하고, 나머지는 Stage 0의 판단 기준으로 자가 점검 후 통과 시 건너뛰십시오;
2. 절약한 시간을 Stage 0의 3단계(회로 및 CAN)와 4단계(시뮬레이션 구축)에 투자하십시오 – 전자는 Stage 1에서 실제 하드웨어를 태우지 않도록 보장하는 안전장치이며, 후자는 물리적 직관 훈련장입니다;
3. 3D 프린팅 상식을 건너뛰지 마십시오. Stage 2의 구조 부품은 프린팅에 크게 의존합니다(Berkeley Humanoid Lite는 사이클로이드 감속기까지 프린팅하며, 출처: `data/roadmap/research/berkeley-humanoid-lite.md`).

### 하드웨어 배경 (기계 / 전자 / 자동화 엔지니어)

당신의 약점은 **소프트웨어 공학과 현대 제어/학습 알고리즘**에 있습니다. 제안:

1. Stage 0 1단계에서 [확률론](/entry/ent_foundation_probability_theory/)과 [볼록 최적화](/entry/ent_foundation_convex_optimization/)를 중점적으로 보충하십시오; 3단계 회로 부분은 30분 만에 개념만 훑어보면 됩니다;
2. Python/ROS 2/git은 반드시 판단 기준에 도달해야 하며, "이해만 하면 된다"는 태도를 버리십시오 – Stage 1부터 모든 작업은 코드 안에서 이루어집니다;
3. 공차, 조립, 안전에 대한 당신의 직관은 귀중한 자산이며, Stage 1(테스트벤치 관절)과 Stage 2(전체 기기 복제)에서 그 가치가 발휘됩니다.

### 기초가 없는 학생

Stage 0 순서대로 성실히 진행하되, **각 단계에 데드라인을 설정**하십시오: 각 절의 "어느 정도까지 배워야 충분한가" 판단 기준을 확인하고, 기준에 도달하면 즉시 넘어가고 뒤돌아 복습하지 마십시오. 제안:

1. 먼저 Wiki [제1장 서론](/wiki/chapters/chapter-01/)을 읽어 전체적인 그림을 그리고, 자신이 어떤 시스템을 구축하고 있는지 이해하십시오;
2. 전체 시간 예산을 ×1.5로 잡고, 예산표의 금액은 변하지 않습니다(Stage 0는 원래 거의 무료입니다);
3. 가장 큰 위험은 "무한 기초 다지기"입니다 – Stage 0의 검수 기준이 이를 방지하기 위한 것입니다. 항목별로 통과하고, 통과 후 즉시 Stage 1로 진입하십시오. 부족한 부분은 Stage 1에서 알려줄 것입니다.

## 카드 페이지의 단계 배지

이 저장소의 카드 페이지(`/entry/...`)에서 `Stage 0 · 기초 기반`과 같은 배지를 볼 수 있습니다. 의미 규칙은 다음과 같습니다:

- **배지 = 학습 시기 힌트**: 해당 항목이 로드맵에서 "가장 먼저 반드시 알아야 하는" 단계입니다. 예를 들어 [URDF 로봇 설명 형식](/entry/ent_technology_urdf_robot_description_format_2024/)은 Stage 0으로 표시되는데, 이를 이해하는 것이 시뮬레이션 입문의 전제 조건이기 때문입니다.
- **한 카드에 여러 배지 가능**: 여러 단계에 걸쳐 반복적으로 사용되는 항목(예: [ROS 2 미들웨어](/entry/ent_software_ros_2_middleware_2024/))은 각각 중요해지는 단계를 표시합니다.
- **배지는 난이도나 중요도 평가가 아님**: Stage 0 항목이 "간단하다"는 뜻이 아니며, Stage 3 항목이 "고급이다"는 뜻도 아닙니다. 단지 로봇 제작 과정에서 나타나는 위치를 나타냅니다.
