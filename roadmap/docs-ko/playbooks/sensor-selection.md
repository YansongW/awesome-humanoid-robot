# 선정 매뉴얼: 센서(Sensor) 어떻게 고를까

> ⚠️ 본 로드맵 내용은 공개 자료와 이론적 지식을 바탕으로 정리되었으며, 실제 기기로 검증되지 않았습니다. 전기 및 기계 작업 시 안전 규정을 직접 확인하시기 바랍니다.

센서 선정 시 흔한 실수는 "완성체 제조사의 홍보 영상대로 갖추는 것"입니다: RealSense를 달고, 6축 힘 센서를 사고, 촉각 어레이도 장착하려 하다가 데이터를 제때 처리하지 못하거나, 구매 후 제어 주파수가 USB 대역폭에 막혀 버리는 경우가 발생합니다. 본 매뉴얼의 접근 방식은 반대입니다: 먼저 로봇이 수행해야 할 작업을 "반드시 알아야 할 물리량"으로 분해한 후, 각 물리량에 대해 충분히 사용 가능한 센서를 선택합니다. 원리 배경은 [제5장 센싱 및 인지 하드웨어](/wiki/chapters/chapter-05/)를, 공급업체 현황은 [제7장 공급업체 지도](/wiki/chapters/chapter-07/)를 참조하십시오.

## 1. 인지 요구사항 분해: 로봇이 반드시 "알아야" 할 것

센서를 세 가지 계층으로 나누고, 각 계층에서 "정말 필요한가?"를 질문합니다:

### 제1계층: 고유 감각(Proprioception) – 관절 각도와 몸체 자세

【할 일】각 능동 관절의 위치 피드백 방식(모터 엔코더, 출력단 엔코더 또는 이중 엔코더)과 몸체/발의 IMU 구성(장착 개수, 샘플링 속도)을 나열합니다.

【이유】고유 감각은 제어 폐루프의 핵심이며, 이것이 없으면 로봇은 "자신의 다리가 어디 있는지"조차 알 수 없습니다. [관절 엔코더 Joint Encoder](/entry/ent_component_joint_encoder_2024/)는 모터 제어에 고해상도 위치 피드백을 제공합니다. 주목할 점은 "어디에 장착하느냐"입니다: 모터 엔코더는 감속기 백래시와 유연 변형을 볼 수 없으므로, 정밀 힘 제어 플랫폼은 출력단 엔코더를 추가하여 이중 엔코더를 구성합니다 – Open Dynamic Robot Initiative의 BLMC 액추에이터가 "이중 엔코더(모터단/출력단)" 설계입니다(출처: 조사 파일 open-dynamic-robot-initiative.md). 저예산方案도 가능합니다: Berkeley Humanoid Lite는 $3의 AS5600 자기 엔코더를 사용하고(출처: 조사 파일 berkeley-humanoid-lite.md), 몸체 자세는 휴대폰급 BNO085 IMU 하나로 제로샷 sim-to-real 보행을 완료했습니다(동일 파일). [IMU](/entry/ent_component_imu_2024/)는 자세 추정을 담당하며, 엔코더 및 비전과 융합하여 고주파 관성 참조를 제공합니다.

【상황 분석 방법】위치 제어 서보 모터 방식(예: Dynamixel)은 엔코더가 내장되어 있어 이 계층은 비용이 들지 않습니다. 자체 개발 QDD 관절은 최소한 모터 엔코더가 필요하며, 출력단 엔코더가 필요한지는 힘 제어를 수행하는지 여부에 달려 있습니다(수행한다면 → 필요). IMU는 몸체급 하나로 시작하면 됩니다. 동적 보행 연구를 한다면 발에 IMU 또는 접촉 센서를 추가하는 것이 좋습니다. BRUCE 이족 로봇이 IMU 샘플링을 2kHz로 한 것도 이 목적입니다(출처: 조사 파일 bruce-westwood.md).

### 제2계층: 힘 감각(Force Sensing) – 말단 힘, 발목 힘, 손끝 촉각

【할 일】세 가지를 결정합니다: 손목/발목에 [6축 힘/토크 센서 Six-axis F/T Sensor](/entry/ent_component_six_axis_force_torque_sensor_2024/)를 장착할지(어느 플랜지에, 범위는 얼마로), [촉각 센서 어레이 Tactile Sensor Array](/entry/ent_component_tactile_sensor_array_2024/)를 장착할지, 아니면 먼저 액추에이터 전류 루프로 힘을 추정하여 대체할지.

【이유】6축 힘 센서는 3방향 힘과 3방향 토크를 동시에 측정하며, 힘 제어 조립, 드래그 티칭, 이족 발목 ZMP 추정에 필수적입니다 – THORMANG3는 양쪽 발목에 ATI Mini58을 각각 장착했습니다(출처: 조사 파일 thormang3.md). 하지만 로봇 전체에서 가장 비싼 센서 중 하나이며, 범위와 정밀도는 서로 상충합니다(범위를 너무 크게 잡으면 분해능이 낭비됩니다). 촉각 어레이는 접촉 압력/전단/질감을 측정하여 정밀 조작의 미끄러짐 감지를 지원합니다. 예산이 부족할 경우 FSR 어레이로 시작할 수 있습니다 – Poppy 휴머노이드 로봇은 2014년에 16개의 FSR 힘 센서를 구성했습니다(출처: 조사 파일 poppy-humanoid.md).

【상황 분석 방법】보행만 하고 조작은 하지 않는다 → 전류 루프 힘 추정 + 발 끝 스위치식 접촉 센서로 충분하며, 6축 힘 센서 비용을 절약합니다. 조립/연마류 조작 연구를 한다 → 손목 6축 힘 센서에 투자할 가치가 있으며, 범위는 "최대 접촉력 × 2"로 선택합니다. 정밀 손 조작 연구를 한다 → 촉각 어레이가 손목 6축 힘 센서보다 우선입니다. 조작 미끄러짐은 주로 손끝에서 발생하기 때문입니다.

### 제3계층: 외부 감각(Exteroception) – 외부 세계 보기

【할 일】외부 감각 작업의 유형을 명확히 합니다: 근거리 조작 장면 이해(0.3–3m), 실내 내비게이션 매핑(3–20m), 또는 실외/대공간. 세 가지는 완전히 다른 센서에 해당합니다.

【이유】[RGB-D 카메라](/entry/ent_component_rgbd_camera/)의 세 가지 깊이 방식 – 구조광, ToF, 스테레오 비전 – 은 각각 적용 가능한 거리와 조명 제약이 있습니다: 구조광은 정밀도가 높지만 강한 빛에 약하고, 스테레오는 수동적이지만 텍스처에 의존합니다. 머리 비전의 주류는 [Intel RealSense](/entry/ent_component_intel_realsense_depth_camera_2024/)(D435i/D455는 Qinglong, G1 등 플랫폼에서 채택, 출처: openloong-qinglong 조사 파일, Unitree 카드)입니다. 내비게이션 매핑의 주력은 라이다이며, [Livox Mid-360](/entry/ent_component_livox_mid_360_lidar_2024/)은 360° 전방위, 265g으로 사족/휴머노이드 SLAM의 인기 선택입니다(출처: Livox 카드).

【상황 분석 방법】데스크탑 조작만 한다 → RGB-D 하나면 충분하며, 라이다는 낭비입니다. 실내 보행 내비게이션을 한다 → RGB-D(근거리 장애물 회피) + 360° LiDAR(매핑)이 성숙한 조합입니다. 순수 비전 연구(VLA, 비주얼 서보) → 예산을 카메라 프레임 속도와 글로벌 셔터에 투자하고, LiDAR를 쌓지 마십시오.

## 2. 부품별 선정표: 핵심 지표 → 로봇에 미치는 영향 → 가격대

### 엔코더 (Encoder)

| 핵심 지표 | 로봇에 미치는 영향 | 가격대 참고 |
|---|---|---|
| 분해능 (비트 수) | 위치 제어 정밀도와 속도 추정 품질 결정; 17비트 이상 관절에서 안정적인 저속 힘 제어 가능 | 자기 엔코더 AS5600 (12비트) 약 $3 (BHL 파일); 고분해능 광전/앱솔루트형은 문의 필요 |
| 앱솔루트 vs 인크리멘탈 | 앱솔루트형은 전원 켜면 각도 인식, 휴머노이드 관절 많아 인크리멘탈형은 매번 전원 켤 때 귀환(초기화)이 번거로움 | 앱솔루트형 비쌈, 필요에 따라 선택 |
| 설치 위치 (모터단/출력단) | 출력단 엔코더는 감속기 백래시와 플렉스플라인 변형을 '볼 수' 있게 하여 힘 제어 투명성의 전제 조건 | 이중 엔코더 = 2배 비용, 힘 제어 플랫폼에 적합 (ODRI 파일) |

### IMU

| 핵심 지표 | 로봇에 미치는 영향 | 가격대 참고 |
|---|---|---|
| 자이로 바이어스 안정성 | 자세 적분 드리프트 속도 결정; [IMU 카드](/entry/ent_component_imu_2024/) 예시 ADIS16475는 2°/hr (운용 중) | 스마트폰급 (BNO085, BHL 사용) 수십 위안대; 산업용 MEMS (ADIS16475급) 가격 미공개로 문의 필요; FOG급 별도 |
| 각 랜덤 워크 (ARW) | 노이즈가 자세 추정에 직접 유입, ADIS16475는 0.15°/√hr | 상동 |
| 축간 정렬 오차 | 다중 센서 융합 시 설치 정렬 작업량, ADIS16475 ±0.1° 출고 시 보정 완료 | 저가형 IMU는 자체 보정 필요 |
| 샘플링 속도/인터페이스 | 균형 제어 루프는 고주파 자세 필요, SPI가 I²C보다 지연 낮음; ADIS16475는 SPI | BRUCE는 2 kHz 구현 (파일) |

### 6축 힘/토크 센서

| 핵심 지표 | 로봇에 미치는 영향 | 가격대 참고 |
|---|---|---|
| 측정 범위 (F/M) | '최대 접촉력 × 2'로 선정; 신위안성 MLL 일반 50/50/100 N, 2/2/4 N·m, [OnRobot HEX-E QC](/entry/ent_component_ati_force_torque_sensor_2024/) ±100/±200 N, ±10 N·m (출처: 두 카드) | 국산 아날로그형 (MLL급) 문의 필요; 수입 ATI Nano25 (±250 N)급 가격 현저히 높음 |
| 정밀도/결합 오차 | 6축 분리 후 힘 제어 신뢰도 결정; MLL 정밀도 ≤0.5% FS, 결합 오차 ≤2% FS | 고정밀 모델 더 비쌈 |
| 과부하 용량 | 양발 발목은 충격 필연적, MLL ≥300% FS, HEX-E 약 500% FS | 과부하 여유 = 수명 |
| 샘플링 속도와 인터페이스 | 힘 제어 루프는 ≥1 kHz 필요; MLL 최대 1000 Hz (수집기 포함), HEX-E는 EtherNet/IP 등 산업용 버스 사용 | 수집기/버스 게이트웨이 별도 |

### 촉각 어레이

| 핵심 지표 | 로봇에 미치는 영향 | 가격대 참고 |
|---|---|---|
| 공간 분해능 (택셀 밀도) | 접촉 형상과 질감 식별 가능 여부 결정, 미끄러짐 사전 예측 지원 | FSR 이산점 가장 저렴 (Poppy 16점 사용); 어레이형 문의 필요 |
| 측정 차원 | 법선 압력만 측정 vs 3축 힘 (전단 포함) – 전단은 미끄러짐 전조 | 3축 어레이 훨씬 비쌈, 파지 연구에만 필요 |
| 곡면 부착과 배선 | 손끝 곡면 설치와 배선은 공학적 주요 난제, 종종 센서 자체보다 공수 더 듬 | 플렉서블 PCB 방식 평가 |

| 핵심 지표 | 로봇에 미치는 영향 | 가격대 참고 |
|---|---|---|
| 이상 측정 범위와 정밀도 | 조작 시나리오는 0.3–1 m 구간 정밀도 중요: [RealSense](/entry/ent_component_intel_realsense_depth_camera_2024/) D435i 0.1–10 m, <2% @2 m; D455 0.6–6 m, <2% @4 m (출처: RealSense 카드, 공식 데이터시트) | D435i 약 USD 199, D455 약 USD 299 (공개 시장 참고, 카드 기준) |
| 셔터 방식 | 머리 움직임 시 글로벌 셔터 (D455)가 롤링 셔터 왜곡 방지 | 일반적으로 글로벌 셔터 모델 약간 비쌈 |
| 내장 IMU | 시각-관성 융합 (VIO)으로 외장 IMU 절약: D435i 내장 BMI055 | i 접미사 모델에만 있음 |
| 연산 의존성 | [ZED 스테레오 카메라](/entry/ent_component_zed_stereo_camera_2024/) 온보드 깊이+공간 AI, 그러나 깊이 계산은 호스트 GPU 생태계 의존 (출처: ZED 카드); 가격은 공급업체에 직접 문의 필요 | RealSense 생태계 (librealsense+ROS) 연산 진입 장벽 낮음, ZED는 NVIDIA 호스트 필요 |

### 라이다 (LiDAR)

| 핵심 지표 | 로봇에 미치는 영향 | 가격대 참고 |
|---|---|---|
| FOV | [Livox Mid-360](/entry/ent_component_livox_mid_360_lidar_2024/) 수평 360°, 수직 -7°~52°, 하나로 상체 주시 가능 (출처: Livox 카드) | Mid-360 약 CNY 3,999 (공식 스토어, 카드 기준) |
| 탐지 거리와 정밀도 | 40 m @10% 반사율, ≤2 cm @10 m – 실내 내비게이션 매핑 충분 | 좌동 |
| 포인트 주파수와 프레임 속도 | 200,000 포인트/초, 10 Hz, 동적 장면 업데이트 속도 결정 | 좌동 |
| 무게/소비 전력 | 265 g, 6.5 W, 머리/상체에 수용 가능; 내장 ICM40609 IMU로 LIO 가능 | 좌동 |

## 3. 캘리브레이션과 시간 동기화: 센서 구매는 절반만 완료

【할 일】캘리브레이션 계획 수립: 카메라 내부 파라미터 (체커보드), 카메라-LiDAR 외부 파라미터, 카메라-IMU 외부 파라미터 및 시간 오프셋, 관절 영점, 그리고 통일된 시간 소스 (PTP 또는 하드웨어 트리거) 약속.

【이유】다중 센서 융합의 모든 알고리즘은 '외부 파라미터 알려짐, 클록 동기화'를 가정합니다. [공동 캘리브레이션 Joint-Camera-IMU Calibration](/entry/ent_method_calibration_joint_camera_imu/) 카드는 표준 절차를 제공합니다: 캘리브레이션 타겟 다중 자세 수집 → 카메라 코너 검출 + LiDAR 평면/코너 추출 → PnP 초기값 + ICP/비선형 최적화 → 재투영 오차 최소화 검증; 캘리브레이션 타겟 없을 시 장면 가장자리 정렬 또는 상호 정보량 기반 온라인 리파인먼트 사용. 시간 오프셋이 제대로 보정되지 않으면 움직임 중 융합 결과는 '고스트 현상' 발생, VIO 발산, 포인트 컬러링 오류 모두 이에서 비롯. 통합 및 테스트 시스템 방법은 [제11장 조립, 통합 및 테스트](/wiki/chapters/chapter-11/) 참조.

【상황별 분석】단일 RGB-D 시작 시 카메라 내부 파라미터만 필요; LiDAR 추가 시 반드시 카메라-LiDAR 외부 파라미터 필요 (카드 방법 그대로 적용 가능); VIO/다중 IMU 융합 시 시간 캘리브레이션 추가. 검증 지표는 재투영 오차 사용, 눈으로 '대충 맞춰졌다'고 판단하지 말 것.

## 4. 두 가지 권장 구성

### 초보자 최소 세트 (목표: 걷기 가능, 볼 수 있음, 고장 안 남)

- 관절 피드백: 스마트 서보 내장 엔코더, 또는 자체 개발 관절에 AS5600급 자기 엔코더 ($3/개, BHL 파일 기준).
- 자세: 스마트폰급 IMU 1개 (BNO085급), 먼저 자세 추정 동작 확인.
- 외부 인식: RealSense D435i 1개 (약 USD 199, 카드 기준) – 깊이+RGB+IMU 일체형, 생태계 가장 성숙.
- 힘 감지: 우선 미사용, 전류 루프로 힘 추정; 발바닥은 스위치/FSR로 지면 접촉 판단.
- 분석 논리: 이 구성은 모든 비용을 '제어 루프 필수 요소'에 집중, 보행과 파지 시연 모두 충분; 병목 현상 명확해지면 (자세 드리프트 큼, 파지 미끄러짐 감지 안 됨) 항목별 업그레이드.

### 연구용 세트 (목표: 힘 제어/조작/내비게이션 논문급 데이터)

- 관절 피드백: 이중 엔코더 (모터단+출력단), 출력단은 고분해능 앱솔루트형.
- 자세: 산업용 MEMS IMU (ADIS16475급: ARW 0.15°/√hr, 바이어스 2°/hr, ±0.1° 정렬, 카드 기준) + 상체/발 분산 배치, 샘플링 1–2 kHz 목표.
- 힘 감지: 손목 6축 힘 (국산 MLL급 또는 ATI/OnRobot급) + 양발 발목 6축 힘 (THORMANG3 구성 선례), 로봇 핸드에 촉각 어레이 추가.
- 외부 인식: RealSense D455 (글로벌 셔터, 약 USD 299) 또는 ZED (GPU 호스트 있을 때) + Livox Mid-360 (약 CNY 3,999)로 360° 매핑.
- 캘리브레이션: 전체 내부/외부 파라미터/시간 동기화 캘리브레이션, 재투영 오차 검증 포함.
- 분석 논리: 연구용 핵심은 '데이터 신뢰도' – 센서 비싼 것이 목적이 아니라, 캘리브레이션 가능, 추적 가능, 클록 동기화가 중요.

## 검증 기준

- '물리량 → 센서 → 설치 위치 → 인터페이스 → 샘플링 속도' 목록 보유, 각 항목에 '없으면 어떻게 되는가' 답변 가능.
- 각 센서의 핵심 지표 (측정 범위/정밀도/대역폭)가 작업 요구사항과 일대일 대응, 카드 또는 데이터시트로 추적 가능; 공개 가격 없는 것은 '공급업체에 직접 문의 필요' 표시.
- 캘리브레이션 계획 실행: 카메라 내부 파라미터 완료, LiDAR 장착 시 카메라-LiDAR 외부 파라미터 완료 및 재투영 오차 기록.
- 전체 센서 타임스탬프 동일 클록 소스로 통일, 융합 노드에서 육안으로 확인 가능한 시간 오차 없음.

## 일반적인 문제점 및 점검 방법

| 증상 | 가능한 원인 | 점검 조치 |
|---|---|---|
| 자세 추정이 느리게 표류하고, 갈수록 틀어짐 | IMU 바이어스 안정성이 부족하거나 온도 보정이 되지 않음 | 바이어스 안정성 지표 확인; 정지 상태에서 바이어스 추정, 필요 시 산업용 IMU로 업그레이드 |
| 보행 중 VIO/융합 발산 | 카메라-IMU 시간 오프셋이 보정되지 않음 | 시간 동기화 보정 수행; 타임스탬프 출처가 통일되었는지 확인 |
| 포인트 클라우드 색상 오정렬, 고스트 현상 | 카메라-LiDAR 외부 파라미터가 오래됨 (충돌 후) | 다중 자세 보정 재실행, 재투영 오차가 수렴하는지 확인 |
| 6축 힘 센서 판독값 제로점 표류 | 온도 드리프트 또는 타레(tare) 미실시; 설치 응력 유입 | 전원 인가 후 예열 후 타레(tare) 실행; 플랜지 설치 예압력 확인 |
| 6축 힘 센서 범위가 항상 포화됨 | "일반 힘" 기준으로 선정했으나 "충격 × 2"가 아님 | 발목 충격 피크값 확인; 더 큰 범위로 교체하거나 과부하 보호 전략 추가 |
| 깊이 이미지가 햇빛 아래에서 넓은 구멍 발생 | 구조광/능동 IR이 강한 빛에 취약 (기술적 한계) | 깊이 기술 경로 확인; 실외에서는 수동 스테레오 또는 LiDAR로 교체 |
| 촉각 어레이 데이터 노이즈가 큼 | 리드선 굽힘 간섭, 전원 공급 리플 | 리드선 고정 배선; 오실로스코프로 전원 공급 확인, 필터링 추가 |
