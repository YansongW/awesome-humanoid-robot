# 阶段 3：完整人形机器人——从双足平台到"听得懂、走得过去、抓得起来"

> ⚠️ 本路线图内容基于公开资料与理论知识整理，未经实机验证；涉及电气与机械操作时请自行核实安全规范。

阶段 2 的双足能稳定行走后，阶段 3 在它之上长出"上半身"：手臂、末端、感知、算力与智能层，完成端到端任务——**听到指令 → 走到桌边 → 抓起杯子**。每步按"做什么 → 为什么 → 你的情况怎么分析"三段式展开，按预算/场景/技能裁剪。

## 3.1 上身与手臂：7 自由度臂与运动学

**步骤 1：定构型与整机模型**
- 【做什么】选定手臂方案：标准每臂 7 自由度（肩 3、肘 1、腕 3），先在仿真建 URDF 确认无干涉，再采购或用阶段 1/2 执行器自制。
- 【为什么】[七自由度机械臂](/entry/ent_component_7dof_arm_2024/) 模仿人臂冗余度：6 自由度只能"到达"位姿，第 7 个让肘部在末端不动时仍能转动，可绕障避奇异。扭矩/自重/价格**需自行向供应商确认**（选型见 [第 4 章](/wiki/chapters/chapter-04/)）。
- 【你的情况怎么分析】只做桌面抓取：先单臂甚至 6 自由度；双手配合、绕障才必须 7 自由度。先仿真再加工金属件。

**步骤 2：运动学与坐标系**
- 【做什么】实现 [正运动学（FK）](/entry/ent_method_forward_kinematics/)（齐次变换链/指数积）与 [逆运动学（IK）](/entry/ent_method_inverse_kinematics/)（Jacobian 伪逆数值解或几何解析解）；7 自由度 IK 在零空间加次级目标（抬肘、远离限位）。再把手臂挂进整机 URDF，统一全身 TF 变换树。
- 【为什么】视觉给出杯子位置后，靠 FK 换算坐标系、靠 IK 求关节角；奇异附近数值 IK 会发散，标准解法是阻尼最小二乘。人形无固定基座，末端精度 = 臂精度 + 躯干姿态误差 + 足端滑移叠加。
- 【你的情况怎么分析】直接用 ROS 生态成熟 IK 求解器；验证顺序：仿真抽查 → 空载 → 带载。行走不稳就"先站定、再操作"。

## 3.2 末端执行器：从夹爪起步

**步骤 3：夹爪起步，预留灵巧手接口**
- 【做什么】首版用电动平行夹爪抓杯子；接口按可替换设计，后续可升级 [LEAP 灵巧手](/entry/ent_component_leap_hand/) 或 [Allegro 灵巧手](/entry/ent_component_allegro_hand/)。
- 【为什么】[灵巧手](/entry/ent_component_dexterous_hand_2024/) vs 夹爪：前者自由度高但控制复杂成本高，后者简单便宜、抓取类型有限。LEAP 是低成本开源 16 自由度拟人手（直驱 Dynamixel + 3D 打印，BOM 需向项目方确认）；Allegro 是 Wonik 商用 16 自由度四指手（扭矩控制关节、ROS 兼容，价格需向供应商确认）。抓杯子类任务，夹爪成功率远高于没调好的灵巧手。
- 【你的情况怎么分析】跑通端到端用夹爪；发操作学习论文从 LEAP 起步；要商用可靠性选 Allegro。

## 3.3 感知栈：RGB-D、激光雷达与标定

**步骤 4：深度相机**
- 【做什么】头部/胸部装一台 [RGB-D 相机](/entry/ent_component_rgbd_camera/) 主视觉，腕部预留近景相机位。主流选型：[Intel RealSense](/entry/ent_component_intel_realsense_depth_camera_2024/) D435i（约 USD 199、量程 0.1–10 m、带 IMU——官方 datasheet）或 D455（约 USD 299、深度精度 <2% @ 4 m）；需 onboard 深度估计考虑 [ZED 立体相机](/entry/ent_component_zed_stereo_camera_2024/)（价格需向供应商确认）。
- 【为什么】深度获取有结构光、ToF、双目立体三条路线（见 RGB-D 卡片）；D435i 最近 0.1 m 适合桌面操作，腕部相机是操作数据的关键来源。
- 【你的情况怎么分析】预算紧一台 D435i 起步；多相机注意 USB 带宽（见常见坑）。

**步骤 5：激光雷达**
- 【做什么】任务含室内导航则装 360° 雷达，如 [Livox Mid-360](/entry/ent_component_livox_mid_360_lidar_2024/)（约 CNY 3,999、265 g、水平 360°、40 m @ 10% 反射率、内置 IMU——官方 datasheet），跑 SLAM 定位。
- 【为什么】RGB-D 量程与 FOV 有限，360° LiDAR 让机器人走动中持续定位绕障。
- 【你的情况怎么分析】房间固定、路线短：RGB-D 里程计即可；跨房间再上 LiDAR。

**步骤 6：联合标定**
- 【做什么】依次做相机内参 → 手眼标定 → 相机—LiDAR—IMU [联合标定](/entry/ent_method_calibration_joint_camera_imu/)：用标定靶最小化重投影误差，ICP 精修点云配准。
- 【为什么】操作本质是坐标变换链：像素 → 相机系 → 基座系 → 末端，任一环外参错了就抓空气。
- 【你的情况怎么分析】先只标头部相机即可跑抓取；标定写成一键脚本，拆装后必须重标。

## 3.4 计算平台：分层算力与实时 Linux

**步骤 7：实时层 + 智能层**
- 【做什么】实时层：工控机/MCU 跑关节控制回路，内核打 [RT-PREEMPT](/entry/ent_software_rt_preempt_linux/) 补丁（使内核大部分可抢占，提供确定性低延迟）。智能层：[Jetson AGX Orin](/entry/ent_component_nvidia_jetson_agx_orin_2024/)（最高 275 TOPS @ 64 GB 版、15–60 W、开发套件约 USD 1,999——NVIDIA 官网/第三方参考）；跑人形基础模型再评估 [Jetson Thor](/entry/ent_component_nvidia_jetson_thor/)（Blackwell 架构、面向 VLA 端侧推理，价格需向供应商确认）。
- 【为什么】控制回路要确定性，AI 推理要吞吐量，混在非实时系统上互相拖累。
- 【你的情况怎么分析】遥操作 + ACT 级策略 Orin 32 GB 够用；VLA/GR00T N1 级推理需 Orin 64 GB 或 Thor。

## 3.5 智能层阶梯：遥操作 → 数据 → 模仿学习 → VLA → 端侧 → 评测

**步骤 8：遥操作**
- 【做什么】搭主从遥操作：参考 [ALOHA](/entry/ent_technology_aloha_teleoperation_system_2023/)（低成本双臂主从硬件，整套约 2 万美元级——见步骤 10 ACT 卡片）；移动场景借鉴 [Mobile ALOHA](/entry/ent_technology_mobile_aloha_2024/) 全身遥操作思路；无整机时用 [UMI 夹爪接口](/entry/ent_technology_umi_gripper_interface_2024/)（手持夹爪 + 相机即可采演示）。
- 【为什么】遥操作一石三鸟：验证硬件可达性、采集模仿学习数据、建立任务 baseline（见 [第 17 章](/wiki/chapters/chapter-17/)）。
- 【你的情况怎么分析】预算极紧走 UMI 先采数据；有双足则遥操作先只管上肢、下肢摇杆定点（Mobile ALOHA 的务实分工）。

**步骤 9：数据采集**
- 【做什么】定义 episode 格式（多视角图像、关节状态、动作、语言指令、时间戳），按统一协议采集并划分训练/验证集。
- 【为什么】格式决定复用性：[Open X-Embodiment](/entry/ent_dataset_open_x_embodiment/) 用统一 RLDS 格式聚合跨本体百万级演示帧，是 VLA 预训练标准语料；[DROID](/entry/ent_dataset_droid/) 示范分布式多实验室采集（局限：只覆盖固定臂）。
- 【你的情况怎么分析】个人采几十到几百条即可训 ACT 级单任务策略；要微调 VLA 就直接对齐 OXE 格式。

**步骤 10：模仿学习**
- 【做什么】顺序试三法：[行为克隆](/entry/ent_method_behavior_cloning/) 做 baseline → [ACT](/entry/ent_method_action_chunking_transformer/)（一次预测约 100 步动作块 + 时间集成平滑，精细双臂任务报告 80%–90% 成功率——见卡片）→ [Diffusion Policy](/entry/ent_method_diffusion_policy/)（去噪扩散建模动作分布，支持多峰动作）。
- 【为什么】逐帧回归会累积复合误差；动作分块与扩散建模是两条主流压制路线（见 [第 18 章](/wiki/chapters/chapter-18/)）。
- 【你的情况怎么分析】第一次做直接上 ACT（与 ALOHA 数据兼容）；动作有多种合法解法选 Diffusion Policy。

**步骤 11：VLA**
- 【做什么】依次评估：[OpenVLA](/entry/ent_method_openvla/)（基于 OXE 训练的开源 VLA）→ [π0](/entry/ent_method_pi0/)（多机器人数据预训练的多面手策略）→ [GR00T N1](/entry/ent_method_gr00t_n1/)（NVIDIA 通用人形基础模型），见 [第 19 章](/wiki/chapters/chapter-19/)。
- 【为什么】VLA 把"听懂指令"从硬编码状态机升级为模型能力：视觉 + 语言直接输出动作。
- 【你的情况怎么分析】无 GPU 集群：OpenVLA 权重 + 自采数据 LoRA 微调。务实路线：VLA 只做指令解析与技能调度，底层抓取由步骤 10 的策略执行。

**步骤 12：端侧部署**
- 【做什么】策略部署到机载 Jetson：[端侧 VLA 推理](/entry/ent_tech_on_device_vla_inference/) 要求模型在内置算力上运行以满足延迟、连接性与隐私约束；手段含量化、推理引擎优化、降输入分辨率。
- 【为什么】移动机器人不能假设网络可用——Wi-Fi 抖动时云端推理等于失控。
- 【你的情况怎么分析】ACT 级小模型 Orin 通常可跑；VLA 先量化测延迟，不达标再降分辨率/减动作块。

**步骤 13：评测**
- 【做什么】仿真用 [LIBERO](/entry/ent_benchmark_libero/)（主题任务套件 + 程序化场景变化的短程桌面基准）回归测试；真机固定任务清单与次数，改动后复测。
- 【为什么】没有基准的调参是玄学：程序化变化暴露过拟合，固定协议复测才能横向比较改动。
- 【你的情况怎么分析】个人：一个 LIBERO 套件 + 真机 20 次固定测试足够。

## 3.6 系统集成与工程化

**步骤 14：中间件**
- 【做什么】以 [ROS 2](/entry/ent_software_ros_2_middleware_2024/)（基于 DDS 的发布/订阅、带实时支持的事实标准中间件）组织全部节点：感知、估计、规划、控制、策略推理分进程通信，冻结接口与 QoS。
- 【为什么】分进程架构让感知崩溃时控制回路仍能安全停机。
- 【你的情况怎么分析】原型期可单 launch 全拉起；联调前必须拆分关键进程并配看门狗。

**步骤 15：安全回路与 OTA**
- 【做什么】（a）装硬件 [急停系统](/entry/ent_component_emergency_stop_system_2024/)：硬接线电路，拍下即切断动力电源或命令安全状态，独立于所有软件；（b）建 [OTA 软件更新](/entry/ent_technology_ota_software_update_2024/) 雏形：无线推送策略/固件/系统软件，带版本管理与回滚。
- 【为什么】急停是全尺寸平台进实验室的准入证——软件急停在死机时不可靠；没有 OTA 每次改策略都要接线。
- 【你的情况怎么分析】摔倒能伤人的平台：急停 + 安全区 + 调试期保险绳缺一不可；桌面小平台可简化为独立硬件电源开关。

## 3.7 参考方案：OpenLoong 与教学级替代

- 【做什么】对照四个有调研档案的项目选参照系（`data/roadmap/research/`，检索时点快照）：

| 项目 | 体型/自由度 | 成本 | 开源与生态 | 适合谁 |
|---|---|---|---|---|
| **OpenLoong/青龙** | 185 cm / 80 kg+，43 主动自由度（含五指灵巧手） | BOM **未知**，公版机不自售 | 主仓库 Apache-2.0；MPC+WBC 可 MuJoCo 零硬件运行；EtherCAT、400 TOPS 主控 | 高校/企业全尺寸二开；个人用其 MuJoCo 框架或关注 NanoLoong |
| **InMoov** | 真人尺寸**上半身**（无腿不能走），约 28 舵机 | 第三方估算 USD 800–2,500 | CC BY-NC 非商业；腱驱五指手；Arduino + MyRobotLab | 低成本体验真人尺寸机器人；学不到行走 |
| **Poppy Humanoid** | 84 cm / 3.5 kg，25 自由度 | 套件约 €9,000 | CC BY-SA / GPLv3；柔顺反驱可示教；主仓库 2021 年停更 | 重教学体系与 HRI 的教育者 |
| **ROBOTIS OP3** | 51 cm / 3.5 kg，20 自由度（XM430-W350，失速扭矩 4.1 N·m） | USD 13,764.35 | 软件 Apache-2.0、2025 复刻版原生 ROS 2；e-Manual 文档标杆 | 预算充足、要开箱即用整机的学校/实验室 |

- 【你的情况怎么分析】机构全尺寸跟 OpenLoong 二开；个人 < USD 1,000 选 InMoov；课堂选 OP3（省心）或 Poppy（体系好但需自维护）。本路线图主线仍是自建，但这些项目的软件栈都可拆用。

## 3.8 整体验收：端到端任务

- 【做什么】把任务拆成五段流水线并联调：指令解析 → 导航至桌边（SLAM + 规划）→ RGB-D 检测杯子并估计位姿 → IK + 抓取规划 → 闭合夹爪并确认。每段单测再整链联调。
- 【为什么】端到端任务是对所有子系统的交叉检验；分段可观测才能定位失败段。
- 【你的情况怎么分析】行走不稳：先以"固定站姿 + 桌面抓取"闭环再解锁行走段；指令解析先用固定句式文本。

## 验收标准

1. 运动学：工作空间内随机抽 100 个目标位姿，IK 求解成功率 ≥95%，真机末端到位误差 ≤2 cm（建议阈值）。
2. 感知：标定后 RGB 点云着色无明显错位（重投影误差 < 2 像素）；杯子 3D 定位误差 ≤1 cm。
3. 智能层：完成 ≥50 条高质量遥操作 episode；真机固定协议下抓取成功率 ≥70%。
4. 端到端：「听到指令 → 走到桌边 → 抓起杯子」连续 10 次试验成功 ≥7 次，单次 ≤2 分钟。
5. 安全：任意时刻拍下急停，动力电源立即切断（实测 <1 s），复位需人工确认。
6. 工程化：手动 kill 感知进程后控制回路不中断，机器人进入安全状态。

## 常见坑与排查

| 症状 | 可能原因 | 排查动作 |
|---|---|---|
| IK 求解失败/关节跳变 | 目标在工作空间外；过奇异位形；限位未建模 | RViz 查目标点；换阻尼最小二乘；核对 URDF 限位 |
| 伸手时整机晃动倾倒 | 未做质心补偿；速度过大 | 降速复测；质心轨迹预规划；站定再操作 |
| ACT/BC 真机远差于训练 | 数据分布偏移；复合误差 | 补采失败场景；查图像与动作时间对齐 |
| VLA 端侧跑不动 | 未量化；显存/算力不足 | 量化测延迟；降输入分辨率；升 Orin 64 GB/Thor |
| 急停拍了不断电 | 急停走了软件路径 | 急停必须硬接线切断动力回路，独立测试 |
| OTA 后起不来 | 无回滚；更新中断电 | A/B 分区；更新前查电量；留串口恢复通道 |
