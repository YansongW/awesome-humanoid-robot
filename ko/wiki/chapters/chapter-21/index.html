제 21 장 데이터 인프라

요약

휴머노이드 로봇의 지능 수준이 '모델 중심'에서 '데이터 중심'으로 전환되고 있다: 시각-언어-행동 모델(Vision-Language-Action, VLA), 모방 학습 및 강화 학습의 성능 상한은 점점 더 순수한 네트워크 구조 혁신이 아닌 훈련 데이터의 규모, 다양성 및 품질에 의해 결정된다. 인터넷 텍스트/이미지와 달리, 로봇 데이터는 웹에서 직접 크롤링할 수 없다. 원격 조작, 모션 캡처, 시뮬레이션 합성 및 로봇 플릿 피드백 등을 통해 '생산'되어야 하며, 정제, 주석, 정렬 및 버전 관리 과정을 거쳐야 훈련 파이프라인에 투입될 수 있다. 본 장은 시스템 엔지니어링 관점에서 휴머노이드 로봇 데이터 인프라의 전체 수명 주기를 설명한다: 데이터 요구사항 모델링 및 모달리티 피라미드, 실제 세계 데이터 수집(원격 조작 시스템, 로봇 없는 수집, 모션 캡처 및 리타겟팅), 시뮬레이션 합성 데이터 및 데이터 엔진, 대표적인 공개 데이터셋 및 벤치마크(Open X-Embodiment, AgiBot World, DROID, ARIO 등), 데이터 형식 및 도구 체인(LeRobot 등), 그리고 스토리지, 대역폭, 비용 및 규정 준수와 같은 엔지니어링 제약 조건을 다루며, 데이터 플릿 스토리지 및 대역폭 추정을 위한 Python 예제를 통해 엔지니어링 규모를 설명한다. 본 장은 하나의 핵심 질문에 답하는 것을 목표로 한다: 범용 휴머노이드 로봇 모델을 지속적으로 공급할 수 있는 '데이터 공장'을 어떻게 설계하고 운영할 것인가? 본 장을 읽은 후, 독자는 모든 데이터 소스에 대해 '비용-충실도-규모'의 3차원 포지셔닝을 수행하고 훈련 파이프라인에 투입하기 전의 엔지니어링 처리 방안을 제시할 수 있어야 한다.

키워드: 데이터 인프라; 데이터 엔진; 데이터 플라이휠; 원격 조작; 모션 캡처; 합성 데이터; 도메인 무작위화; Open X-Embodiment; AgiBot World; DROID; LeRobot; VLA

21.1 데이터 인프라 개요

21.1.1 모델 중심에서 데이터 중심으로: 휴머노이드 로봇의 '데이터 병목'

고전 로봇 공학에서는 행동이 주로 정밀한 모델링과 최적화 제어를 통해 생성되었습니다. 운동 방정식, 제약 조건 및 비용 함수는 엔지니어가 수동으로 구성했으며, 시스템의 일반화 능력은 모델의 정밀도에 의해 제한되었습니다. 반면, 학습 중심의 새로운 패러다임에서는 행동이 대규모 시연 데이터 또는 상호 작용 데이터로부터 '증류'됩니다. 확산 정책(Diffusion Policy), 동작 청킹 트랜스포머(Action Chunking Transformer, ACT) 및 다양한 VLA 모델(예: OpenVLA, π0, GR00T N1)은 모두 방대한 양의 다중 모드 궤적을 훈련 원료로 사용합니다.

이러한 패러다임 전환은 구조적 모순을 야기합니다. 강조해야 할 점은 '데이터 중심'이 모델 지식의 소멸을 의미하지 않는다는 것입니다. 운동 모델, 기하학적 제약 조건 및 안전 규칙은 여전히 데이터 파이프라인의 품질 검사, 재지정 및 실행 가능성 선별 단계에서 기본적인 역할을 수행하며, 변화는 단지 '행동의 직접 생성자'에서 '데이터의 관문'으로 역할이 바뀌었을 뿐입니다.

구체적으로, 구조적 모순은 다음 세 가지 측면에서 나타납니다.

  • 인터넷 데이터는 '무료'이지만, 로봇 데이터는 '비싸다'. 텍스트와 이미지는 공개 네트워크에서 매우 낮은 비용으로 얻을 수 있지만, 관절 상태, 힘 감각 및 다중 시점 비디오가 포함된 로봇 조작 궤적은 실제 하드웨어 또는 고충실도 시뮬레이션에서 초 단위로 '생산'되어야 합니다.
  • 휴머노이드 로봇의 데이터 부족은 특히 심각합니다. 산업용 로봇 암은 수십 년간의 배치 경험이 축적되어 있지만, 범용 휴머노이드 로봇의 보유량은 최근에야 만 대 수준에 도달했습니다. 또한 휴머노이드 로봇은 자유도가 높고(전신 30–60개 관절), 작업 범위가 넓으며(보행, 운반, 양손 조작, 인간-로봇 상호 작용), 단일 궤적의 정보 차원이 고정 로봇 암보다 훨씬 높습니다.
  • 데이터 분포가 능력의 경계를 결정합니다. 행동 복제(Behavior Cloning)의 일반화 오류는 시연 데이터가 목표 분포를 얼마나 잘 덮는지에 따라 달라집니다. 단일 실험실, 단일 본체, 단일 작업에서 수집된 데이터로 훈련된 정책은 개방 환경으로 전이되기 어렵습니다.

데이터 분포의 중요성은 공변량 이동(covariate shift) 관점에서 정량적으로 이해할 수 있습니다. 각 시간 단계에서 정책의 오류 확률을 \(\epsilon\)이라고 하고, 작업 시간 영역을 \(T\)라고 하면, 행동 복제의 누적 오류는 \(T\)에 따라 초선형적으로 증가합니다. 고전적 분석은 상한을 \(O(\epsilon T^2)\)으로 제시합니다. 한 단계의 오류는 로봇을 훈련 분포 외부의 상태로 이끌고, 다음 단계의 오류 확률은 더욱 증가하여 '오류 복합' 효과를 형성합니다. DAgger(Dataset Aggregation) 계열 방법은 전문가가 정책이 실제로 방문한 상태에 대해 반복적으로 레이블을 지정하도록 하여 상한을 \(O(\epsilon T)\)로 낮춥니다. 그 본질은 방향성 있는 추가 데이터 수집을 통해 분포 이동을 수정하는 것입니다. 이것이 바로 데이터 엔진 폐쇄 루프(21.3.3절 참조)의 이론적 근거입니다.

따라서 업계에서는 일반적으로 데이터 인프라를 액추에이터, 배터리와 동등하게 중요한 '전략적 자산'으로 간주합니다. 더 낮은 단위 비용과 더 높은 처리량으로 고품질 다중 모드 데이터를 생산할 수 있는 기업이 모델 반복의 주도권을 쥐게 됩니다. 이 폐쇄 루프 논리는 데이터 플라이휠(Data Flywheel)이라고 비유적으로 불립니다. 배치된 로봇이 데이터를 생성하고, 데이터가 모델을 개선하며, 개선된 모델이 로봇의 능력과 작업 성공률을 향상시켜 더 큰 규모의 배치와 더 많은 데이터를 지원합니다.

용어 설명: 데이터 중심, 데이터 플라이휠, VLA

  • 데이터 중심(data-driven): 시스템 행동이 주로 데이터로부터 학습된 모델에 의해 결정되며, 수동으로 구축된 해석적 모델에 의해 결정되지 않습니다.
  • 데이터 플라이휠(data flywheel): 로봇 배치가 데이터를 생성하고, 데이터가 AI 모델을 개선하며, 모델이 로봇 성능을 향상시키고 더 많은 데이터를 생성하는 자기 강화 순환입니다.
  • VLA(Vision-Language-Action model): 시각적 관찰과 언어 명령을 입력으로 받아 로봇 동작을 직접 출력하는 대규모 모델로, 현재 범용 휴머노이드 로봇 지능의 주요 기술 경로 중 하나입니다.

21.1.2 데이터 모드와 데이터 피라미드

휴머노이드 로봇 훈련 데이터는 '획득 비용-규모-충실도'의 세 가지 차원에서 자연스럽게 피라미드 구조를 형성합니다.

flowchart TB
    A[인터넷 비디오 및 이미지<br/>규모 매우 큼 · 비용 매우 낮음<br/>동작 레이블 없음] --> B[인간 중심 비디오<br/>Ego4D / Epic-Kitchens<br/>1인칭 시점 · 로봇 본체 정보 부족]
    B --> C[인간 모션 캡처 데이터<br/>AMASS / OptiTrack 수집<br/>전신 자세 있음 · 재지정 필요]
    C --> D[시뮬레이션 합성 데이터<br/>Isaac Sim / MuJoCo / Genesis<br/>규모 제어 가능 · Sim-to-Real 격차 존재]
    D --> E[실제 로봇 원격 조작 데이터<br/>ALOHA / Mobile ALOHA / 차량 회귀<br/>충실도 가장 높음 · 비용 가장 높음]

각 계층은 대체 관계가 아닌 상호 보완 관계입니다. 현대 훈련 파이프라인은 일반적으로 '피라미드형 커리큘럼'을 채택합니다. 먼저 인터넷 비디오와 이미지로 시각-언어 표현을 사전 훈련하고, 인간 비디오와 모션 캡처 데이터로 작업 사전 지식과 전신 운동 사전 지식을 학습하며, 대규모 시뮬레이션 데이터로 하위 수준 제어 및 정책을 훈련하고, 마지막으로 소량의 고품질 실제 로봇 데이터로 정렬 및 미세 조정을 수행합니다. HumanPlus 섀도우 시스템(HumanPlus Shadowing System)은 이 접근 방식의 대표적인 예입니다. 먼저 AMASS와 같은 인체 동작 데이터를 사용하여 시뮬레이션에서 전신 하위 수준 정책을 훈련한 다음, RGB 카메라 인체 자세 추정을 통해 실제 휴머노이드 로봇을 구동하여 조작 데이터를 수집합니다.

모드 관점에서 볼 때, 완전한 휴머노이드 로봇 훈련 샘플(일반적으로 궤적 또는 에피소드(episode)라고 함)에는 일반적으로 다음이 포함됩니다.

모드 일반적인 내용 일반적인 샘플링 속도 주요 용도
시각 머리/손목/3인칭 RGB, 깊이 15–60 Hz 장면 이해, 정책 입력
본체 상태 관절 각도, 관절 속도, IMU, 베이스 자세 100–1000 Hz 상태 추정, 정책 입력, 동작 감독
동작 레이블 관절 목표 위치/토크, 말단 자세 30–500 Hz 행동 복제의 감독 신호
힘/촉각 6축 힘 센서, 촉각 어레이 100–1000 Hz 접촉이 많은 작업, 안전 모니터링
언어 작업 명령, 단계별 설명, 음성 이벤트 수준 VLA 조건 입력, 작업 레이블링
환경 컨텍스트 오디오, 장면 포인트 클라우드, 객체 자세 실제값 1–30 Hz 다중 모드 이해, 데이터 선별

용어 설명: 에피소드, 다중 샘플링 속도 혼합, 동작 레이블

  • 에피소드(episode): 한 번의 완전한 작업 실행 기록 단위로, 로봇 학습 데이터의 기본 단위입니다. 하나의 에피소드 내에는 수십에서 수천 개의 시간 단계가 포함됩니다.
  • 다중 샘플링 속도 혼합: 서로 다른 모드의 샘플링 속도 차이는 최대 두 자릿수(예: 30 Hz 비디오 vs. 1 kHz 토크)에 달할 수 있습니다. 저장 형식은 타임스탬프 정렬 검색을 지원해야 하며, 훈련 시 일반적으로 가장 낮은 주파수 모드를 기준으로 정렬 샘플링하거나 고주파 모드에 대해 다운샘플링 및 특징 풀링을 수행합니다.
  • 동작 레이블(action label): 지도 학습에서 '정답' 역할을 하는 동작 신호입니다. 원격 조작 데이터에서는 운영자의 명령 채널에서 직접 가져오고, 시뮬레이션 데이터에서는 생성 정책에서 가져오며, 로봇 수집 데이터가 없는 경우 재지정 후처리를 통해 간접적으로 얻으며, 품질은 후처리 체인의 정밀도에 따라 달라집니다.

21.1.3 데이터 인프라의 전체 수명 주기

데이터 인프라는 단일 '데이터 세트 저장소'가 아니라 생산, 유통, 소비 및 회귀를 포괄하는 완전한 파이프라인입니다. 그 수명 주기는 6단계로 나눌 수 있습니다.

flowchart LR
    A[수집<br/>원격 조작/모션 캡처/시뮬레이션] --> B[전송 및 데이터 레이크 적재<br/>에지 캐싱/네트워크 회귀]
    B --> C[저장 및 버전 관리<br/>객체 스토리지/데이터 레이크]
    C --> D[정제 및 레이블링<br/>정렬/품질 검사/의미 레이블링]
    D --> E[훈련 및 평가<br/>데이터 비율/샘플링]
    E --> F[배치 및 회귀<br/>OTA 배포/실패 사례 회귀]
    F -->|실패 및 긴 꼬리 샘플| A

각 단계에는 고유한 엔지니어링 제약 조건이 있습니다. 수집 단계는 하드웨어 수와 운영자 효율성에 의해 제한됩니다. 전송 단계는 대역폭과 에지 저장소에 의해 제한됩니다. 저장 단계는 비용과 검색 효율성에 의해 제한됩니다. 레이블링 단계는 인력과 자동화 도구에 의해 제한됩니다. 훈련 단계는 데이터 비율과 커리큘럼 설계에 관심이 있습니다. 배치 단계는 OTA 소프트웨어 업데이트를 통해 새 모델을 배포하고 실패 사례와 긴 꼬리 시나리오를 회귀시켜 차량 데이터 플라이휠(Fleet Data Flywheel)을 형성합니다. 21.2–21.5절에서는 이 파이프라인을 따라 하나씩 설명합니다.

조직 관점에서 볼 때, 이 파이프라인은 일반적으로 네 가지 유형의 팀에 걸쳐 있습니다. 수집 운영(작업장 및 운영자 관리), 플랫폼 엔지니어링(저장, 전송 및 도구 체인), 알고리즘(훈련, 평가 및 비율 결정) 및 규정 준수 및 보안입니다. 데이터 인프라 프로젝트의 가장 일반적인 실패 모드는 기술 선택 오류가 아니라 네 팀이 '어떤 데이터가 유용한지'에 대한 공통된 측정 기준이 부족하다는 것입니다. 실제 로봇 평가를 기준점으로 하는 통합 지표 시스템을 구축하는 것이 프로젝트 거버넌스의 첫 번째 과제입니다.

21.1.4 이 장과 다른 장의 관계

데이터 인프라는 하드웨어 계층과 알고리즘 계층 사이에 위치합니다. 5장에서 논의된 센싱 및 컴퓨팅 하드웨어는 데이터 수집의 물리적 상한(카메라 사양, 버스 대역폭, 에지 컴퓨팅 성능)을 결정합니다. 22장의 소프트웨어 미들웨어는 로봇 내 데이터 흐름 방식(ROS 2 토픽, DDS QoS, 시간 동기화)을 결정합니다. 그리고 후속 장의 인식, 계획, 제어 및 VLA 모델은 데이터 파이프라인의 최종 소비자입니다. 이 장을 이해하는 한 가지 주요 라인은 다음과 같습니다. 데이터 인프라의 설계 지표(처리량, 동기화 정밀도, 유효 데이터 비율, 단위 비용)는 궁극적으로 하류 모델의 실제 로봇 성능 향상으로 환산되어야 합니다. 모델 평가 없이 데이터 규모에 대해 논하는 것은 의미가 없습니다.

21.2 실제 세계 데이터 수집

실제 세계 데이터는 휴머노이드 로봇 훈련 데이터 중 충실도가 가장 높고 단위 비용도 가장 높은 유형입니다. 이 절에서는 "로봇 유/무/인체 측/군집 측" 네 가지 기술 경로에 따라 설명합니다: 원격 조작 시스템은 직접 "관측-행동" 쌍을 생성하고, 로봇 없는 수집은 인간 시연을 통해 하드웨어 병목을 우회하며, 모션 캡처는 전신 움직임의 실제 값을 제공하고, 군집 환류는 배치 자체를 지속적인 데이터 소스로 전환합니다. 네 가지 경로의 비용 구조, 데이터 특성 및 적용 단계는 각각 다르며, 실제 프로젝트는 일반적으로 조합하여 사용합니다.

21.2.1 원격 조작 수집: 주종형 원격 조작 시스템

실제 로봇 데이터 중 가장 품질이 높은 유형은 원격 조작(teleoperation)에서 비롯됩니다: 인간 조작자가 주 장치를 통해 종 로봇을 직접 제어하여 작업을 완료하고, 시스템은 모든 관측과 행동을 동기화하여 기록하며, 자연스럽게 "관측-행동" 쌍을 형성하여 행동 복제에 직접 사용할 수 있습니다.

이 방향의 대표적인 시스템은 ALOHA 원격 조작 시스템(ALOHA Teleoperation System)입니다: 저비용 양팔 주종 원격 조작 하드웨어로, 조작자가 로봇을 등지고 주 팔로 종 팔을 직접 견인하여 움직이며, 기계적 결합을 통해 직관적인 힘 감각과 운동 매핑을 얻어 양손 정밀 조작의 모방 학습 데이터를 수집합니다. 파생 플랫폼인 Mobile ALOHA는 양팔을 이동 베이스에 장착하여, 조작자가 전신 견인을 통해 요리, 청소, 엘리베이터 타기 등 "이동 + 양손 조작" 복합 능력이 필요한 작업을 완료하며, 가정 및 창고 시나리오 데이터를 수집합니다. ALOHA 시리즈는 두 가지 중요한 공학적 결론을 증명했습니다:

  • 저비용 하드웨어로 고가치 데이터를 생산할 수 있습니다. ALOHA는 수만 위안 수준의 하드웨어 비용으로 이전에는 고가의 힘 제어 로봇 팔이 필요했던 접촉이 풍부한 조작 데이터 수집을 실현했습니다.
  • 주종 동형 설계는 인지 부담을 현저히 줄입니다. 주 팔과 종 팔의 운동학이 일치할 때, 조작자는 장기 훈련 없이도 높은 수집 효율과 성공률을 달성할 수 있습니다.

보다 일반적인 양방향 원격 조작(Bilateral Teleoperation) 아키텍처에서는 명령 신호가 주 측에서 종 측으로 흐르고, 힘/촉각 피드백이 종 측에서 주 측으로 반환되어 투명성과 안전성을 향상시키며, 특히 깨지기 쉬운 물체 조작, 조립 등 접촉에 민감한 시나리오에 적합합니다. 그 대가는 시스템 복잡성과 비용 증가, 그리고 통신 지연에 대한 엄격한 요구사항입니다.

주종 로봇 팔 외에도 데이터 팩토리에서 흔히 볼 수 있는 원격 조작 인터페이스는 다음과 같습니다: VR 헤드셋과 컨트롤러/데이터 글러브(몰입형 1인칭 시점과 6자유도 손 추적 제공, 휴머노이드 로봇 전신 원격 조작에 적합), 외골격(조작자의 전신 관절 각도를 직접 측정하여 로봇 전신에 매핑). 어떤 인터페이스든 종단 간 지연(end-to-end latency)은 핵심 경험 지표입니다: 일반적으로 시각 피드백과 제어 루프의 총 지연이 약 100ms를 초과하면 조작자의 제어 정밀도와 작업 성공률이 현저히 떨어지므로, 수집 시스템은 일반적으로 클라우드 중계를 거치지 않고 로컬 직접 연결 비디오와 저지연 이미지 전송을 사용합니다.

용어 설명: 원격 조작, 주종 동형, 양방향 원격 조작, 투명성

  • 원격 조작(teleoperation): 인간 조작자가 인터페이스 장치를 통해 로봇을 원격 제어하며, 로봇 상태와 조작 명령이 동기화되어 시연 데이터로 기록됩니다.
  • 주종 동형(isomorphic leader-follower): 주 장치와 종 로봇의 운동학 구조가 일치하여 관절 공간에 직접 매핑이 존재합니다.
  • 양방향 원격 조작(bilateral teleoperation): 주종 제어에 종 측에서 주 측으로의 힘/촉각 피드백 채널을 추가합니다.
  • 투명성(transparency): 조작자가 느끼는 힘이 종 측의 실제 접촉 힘과 일치하는 정도로, 원격 조작 시스템 충실도의 핵심 지표입니다.

원격 조작 수집의 효율은 "시간당 유효 궤적 수"로 설명할 수 있습니다. 단일 작업 평균 실행 시간을 \(T_{task}\), 작업 간 재설정 시간을 \(T_{reset}\), 성공률을 \(p\)라고 하면, 단일 수집 워크스테이션의 유효 처리량은 다음과 같습니다.

\[ \eta = \frac{3600 \cdot p}{T_{task} + T_{reset}} \quad [\text{개/시간}] \]

일반적으로 데스크톱 수준의 양손 조작 작업의 \(T_{task}\)는 30–120초 사이이고, \(T_{reset}\)은 종종 10–60초(물체 재배치, 장면 정리)가 필요하며, 훈련되지 않은 조작자의 성공률 \(p\)는 약 60–80%입니다. 이는 단일 워크스테이션의 일일 생산량이 일반적으로 수백 개 수준에 불과하다는 것을 의미합니다. 수십만 개 수준의 데이터 세트를 축적하려면 수십에서 수백 개의 수집 워크스테이션을 병렬로 배포해야 하며, 이것이 바로 "데이터 팩토리" 모델의 유래입니다.

21.2.2 수집 프로토콜 설계: 작업, 메타데이터 및 품질 관리

데이터 팩토리와 일반 실험실 수집의 근본적인 차이는 프로토콜화에 있습니다. 확장 가능한 수집 프로토콜은 최소한 다음 요소를 포함해야 합니다:

  • 작업 분해 규범: 복잡한 작업을 명확한 시작/종료 기준이 있는 원자 단위(episode)로 분해하며, 각 단위는 하나의 궤적에 해당합니다. 작업 설명은 구조화된 템플릿(장면, 물체, 목표 상태, 성공 기준)을 사용하여 이후 검색과 매칭을 용이하게 합니다.
  • 메타데이터 규범: 각 궤적은 조작자 ID, 워크스테이션 ID, 본체 번호 및 캘리브레이션 버전, 수집 소프트웨어 버전, 환경 태그(조명, 날씨, 장면 유형)와 바인딩됩니다. 메타데이터가 누락된 궤적은 대규모 혼합 훈련 시 문제 출처를 추적하기 거의 불가능합니다.
  • 시간 동기화 규범: 모든 카메라, 본체 상태 및 힘 신호는 통합 클록 소스(하드 동기화 또는 PTP/NTP 소프트 동기화)를 공유하며, 저장 전에 타임스탬프 정렬 검증을 수행합니다.
  • 품질 검사 프로세스: 수집 측 실시간 품질 검사(프레임 손실, 동기화 오차, 캘리브레이션 실패 자동 경고)와 오프라인 샘플링 검사(워크스테이션 및 조작자별 계층 샘플링을 통한 성공률 레이블 재확인)를 포함합니다.

프로토콜화의 이점은 복합적입니다: 통합 프로토콜을 통해 여러 워크스테이션, 여러 도시, 심지어 여러 국가의 수집 생산 능력을 하나의 데이터 레이크로 통합할 수 있습니다. 구조화된 메타데이터는 "작업 분포별 샘플링 훈련"을 가능하게 하며, 이는 VLA 훈련 효과의 핵심 변수 중 하나입니다.

21.2.3 로봇 없는 수집: 휴대형 인터페이스 및 1인칭 시점 데이터

원격 조작의 병목은 로봇 하드웨어 수량에 있습니다. 한 가지 대안적인 접근 방식은 로봇 없는 수집(robot-free data collection)입니다: 인간이 센서가 부착된 도구를 손에 들고 직접 작업을 완료한 후, 인간의 동작을 로봇 본체에 매핑하는 것입니다.

  • UMI 그리퍼 인터페이스(UMI Gripper Interface): 휴대형 그리퍼와 손목 카메라의 조합으로, 조작자가 자연 환경에서 그리퍼를 손에 들고 파지 및 조작을 수행하며, 시스템은 1인칭 시점 비디오와 그리퍼 개폐 상태를 기록한 후, 후처리를 통해 6자유도 자세 궤적을 복원합니다. 그 장점은 로봇 본체에 대한 의존성을 완전히 제거하여, 임의의 실제 환경에서 대규모 병렬 "야외" 데이터를 수집할 수 있다는 점입니다.
  • HuDOR: 인간에서 로봇으로의 손 시연을 위한 프레임워크로, 인간 손의 정밀 동작 데이터를 수집하여 로봇 손의 모방 학습 전략을 훈련하며, 고자유도 손 원격 조작의 어려움을 해결합니다.
  • 인간 중심 비디오 데이터 세트: Ego4D 1인칭 시점 비디오 데이터 세트(3000시간 이상의 1인칭 시점 비디오, 1인칭 기술 및 상호 작용 이해용)와 Epic-Kitchens 데이터 세트(1인칭 시점 주방 상호 작용 비디오)는 로봇 동작 레이블을 포함하지 않지만, affordance(사용 가능성), 손-물체 상호 작용 패턴 및 작업 시간 구조를 학습하는 데 널리 사용되어 전략에 "인간 사전 지식"을 제공합니다.

로봇 없는 수집의 핵심 기술적 과제는 교차 구현 격차(embodiment gap)입니다: 인간 손과 로봇 그리퍼/로봇 손은 기하학, 자유도 및 접촉 특성에서 체계적인 차이가 있으며, 직접 매핑하면 실행 불가능한 동작이 발생합니다. 공학적으로는 일반적으로 동작 리타겟팅, 파지 매핑 및 실행 가능성 필터링과 같은 후처리 단계가 필요하며, 소량의 실제 로봇 데이터를 사용하여 정렬을 수행합니다.

데이터 경제학적 관점에서 볼 때, 로봇 없는 수집은 "데이터 생산"을 자본 집약적(로봇 군집 구매 및 유지 관리)에서 노동 집약적(수집가 모집 및 훈련)으로 전환하며, 그 한계 비용 구조는 전통적인 데이터 레이블링 산업에 더 가깝습니다. 따라서 특히 목표 배포 환경(가정, 상점, 창고)에서 현장 수집에 적합합니다. 수집가가 휴대용 장치를 가지고 실제 장면에 들어가면 장면의 현실성과 동작의 자연스러움을 동시에 얻을 수 있으며, 이는 어떤 실험실 워크스테이션도 복제할 수 없습니다.

21.2.4 모션 캡처 및 전신 동작 리타겟팅

걷기, 전신 협응과 같은 운동 능력의 경우, 모션 캡처(motion capture, mocap)는 고품질 전신 움직임 데이터를 얻는 주요 수단입니다. OptiTrack 모션 캡처 시스템은 적외선 광학 마커를 사용하여 서브밀리미터 수준의 3차원 인체 추적을 실현하며, 실험실 수준의 휴머노이드 로봇 움직임 데이터 수집 및 로봇 리타겟팅의 대표적인 장비입니다.

데이터 자산 측면에서, AMASS 동작 데이터 세트는 여러 공개 모션 캡처 데이터베이스를 통합하고, 통합 파라미터화(SMPL 인체 모델)를 통해 대규모의 깨끗한 인체 전신 자세 및 형태 시퀀스를 형성하여, 휴머노이드 로봇 시뮬레이션 사전 훈련에 널리 사용됩니다: 먼저 AMASS에서 저수준 전신 추적 전략을 훈련한 후, 실제 로봇으로 전이합니다. HumanPlus Shadowing 데이터 세트는 한 걸음 더 나아가, 휴머노이드 로봇이 인체 움직임을 "그림자처럼 따라가며" 직접 수집하여, 인체 움직임과 로봇 본체 상태를 바인딩하고, 전신 모방 및 원격 조작 전략을 훈련하는 데 사용됩니다.

모션 캡처 경로의 비용 구조는 원격 조작과 시뮬레이션 사이에 있습니다: 광학 모션 캡처 장소와 장비 투자가 크고, 전문가의 유지 보수 및 캘리브레이션이 필요하지만, 단일 전신 움직임 수집은 단 몇 초면 완료되며, 단위 시간당 데이터 밀도가 매우 높습니다. 주요 한계는 조작 작업에 필요한 물체 상호 작용 및 힘 정보가 부족하다는 점이므로, 일반적으로 원격 조작 데이터와 상호 보완적으로 사용되며, 서로 대체되지는 않습니다.

인체에서 로봇으로의 동작 리타겟팅(motion retargeting)은 일반적으로 제약 조건이 있는 최적화 문제로 표현됩니다: 인체와 로봇의 관절 구조가 다르고 링크 길이가 다른 조건에서, 로봇 관절 궤적 \(q(t)\)를 찾아 주요 작업 공간 특징(손/발 위치, 질량 중심, 방향)이 인간 시연을 추적하도록 합니다:

\[ \min_{q(t)} \sum_{k} w_k \left\| f_k(q(t)) - x_k^{human}(t) \right\|^2 \quad \text{s.t.} \quad q_{min} \le q \le q_{max},\; |\dot{q}| \le \dot{q}_{max} \]

여기서 \(f_k(\cdot)\)는 \(k\)번째 작업 공간 특징의 순운동학 매핑이고, \(w_k\)는 가중치입니다. 공학적으로는 자체 충돌, 지면 접촉 일관성 및 동역학적 실행 가능성(질량 중심 투영이 지지 영역 내에 있어야 함)을 처리해야 합니다. 그렇지 않으면 리타겟팅된 움직임이 실제 로봇에서 실행 불가능할 수 있습니다.

보행 운동의 경우, 접촉 타이밍 일관성을 추가로 확인해야 합니다. 인간 시연에서의 지지기/스윙기 전환 시점은 재지정 후 로봇의 발바닥 접촉력 일관성과 일치해야 합니다. 만약 인간이 단일 지지기에서 양쪽 지지가 필요한 동작을 수행했다면, 직접 재지정 시 불균형이 발생합니다. 공학 실무에서는 일반적으로 접촉 상태를 위 최적화 문제에 강한 제약 조건 또는 강한 정규화 항으로 추가하고, 시뮬레이션에서 전신 동역학 롤아웃을 통해 최종 실행 가능성을 필터링하며, 이 필터링을 통과한 세그먼트만 실제 로봇 훈련 세트에 포함됩니다.

21.2.5 차량 회귀와 데이터 플라이휠

로봇이 대규모 배치 단계에 진입하면, 데이터 수집의 주요 전장은 "데이터 공장"에서 "운용 중인 차량"으로 전환됩니다. 차량 데이터 플라이휠(Fleet Data Flywheel)은 다음과 같은 폐쇄 루프를 설명합니다. 고객 현장에 배치된 로봇이 작업 실행 과정(특히 실패, 수동 개입 및 낮은 신뢰도 세그먼트)을 지속적으로 기록하고, 필터링 후 클라우드로 전송하여 다음 모델 훈련에 사용됩니다. 새 모델은 OTA 소프트웨어 업데이트를 통해 배포되어 차량 전체의 성공률을 높이고, 더 많은 작업 시나리오를 해제하며 더 많은 데이터를 생성합니다.

차량 회귀 데이터의 독특한 가치는 다음과 같습니다.

  • 실제 롱테일 분포. 데이터 공장의 작업장 시나리오가 아무리 다양하더라도 실제 배치에서의 조명, 지면, 물체 및 인간-로봇 상호작용의 롱테일을 완전히 커버하기 어렵습니다. 바로 이러한 롱테일 샘플이 정책의 강건성을 결정합니다.
  • 실패 샘플은 감독 신호. 수동 개입(intervention) 시점은 정책의 실패 경계를 자연스럽게 표시하며, 오프라인 데이터에 대한 중요한 보완입니다.
  • 한계 비용 감소. 수집 하드웨어는 이미 고객이 비용을 지불하여 배치했으므로, 데이터의 한계 비용은 주로 대역폭, 저장 및 레이블링이며, 자체 구축 수집 차량보다 훨씬 낮습니다.

그러나 공학적 제약도 명확합니다. 개인정보 보호 규정(현장에 얼굴, 가정 환경 등 민감 정보 포함 가능), 회귀 대역폭(다중 비디오 데이터 양이 방대하여 일반적으로 엣지에서 사전 필터링 필요), 그리고 데이터 버전과 모델 버전의 추적 가능성(어느 버전의 모델이 어떤 데이터로 훈련되었는지) 등이 있습니다. 21.5절에서 정량적인 공학적 추정을 제공합니다.

21.3 시뮬레이션 합성 데이터와 데이터 엔진

실제 데이터는 물리적 시간과 하드웨어 수량의 제약을 받지만, 시뮬레이션은 데이터 생성을 무한 병렬 컴퓨팅 문제로 전환합니다. 이 절에서는 시뮬레이션 데이터 파이프라인의 구성, 데모 증강 및 자동 작업 생성이라는 두 가지 증폭 기술, 그리고 Sim-to-Real 격차라는 근본적인 제약 조건에 대해 논의합니다. 바로 이 제약 조건이 시뮬레이션 데이터를 실제 데이터와 혼합하여 사용해야 하며, "데이터 엔진"이 개방 루프가 아닌 폐쇄 루프여야 하는 이유를 결정합니다.

21.3.1 시뮬레이션 데이터 생성 파이프라인

실제 데이터는 비싸고 물리적 시간의 제약을 받습니다. 시뮬레이션 합성 데이터(synthetic data)는 병렬화 및 확장 가능한 두 번째 소스를 제공합니다. 현재 주류 시뮬레이션 및 훈련 플랫폼은 다음과 같습니다.

시뮬레이션 데이터 파이프라인의 일반적인 흐름은 다음과 같습니다. 장면 및 에셋 생성(물체, 조명, 텍스처) → 작업 정의(성공 기준, 초기 조건 분포) → 정책 또는 플래너 롤아웃(스크립트 정책, 최적화 컨트롤러 또는 훈련된 정책에 의한 데모 생성) → 데이터 내보내기 및 품질 검사. 시뮬레이션은 완벽한 실제 값(물체 자세, 분할 마스크, 접촉력)을 제공할 수 있으므로, 합성 데이터는 집중적인 주석이 필요한 작업에서 자연스러운 이점을 가집니다.

실제 수집과 비교할 때, 시뮬레이션 데이터 생산의 경제 모델은 완전히 다릅니다. 처리량은 컴퓨팅 성능과 거의 선형적으로 상관관계가 있습니다. GPU 병렬 인스턴스 수 \(N_{env}\)가 일일 생산량을 결정하며, 단일 데이터의 한계 비용은 주로 전기 요금과 컴퓨팅 성능 감가상각으로, 일반적으로 원격 조작보다 1~2자릿수 낮습니다. 그러나 이 이점은 "정책이 시뮬레이션에서 실제로 전이 가능한 기술을 학습할 수 있을 때"만 유효하며, 이것이 21.3.3절에서 논의할 Sim-to-Real 제약 조건입니다.

그 중 두 가지 요소가 합성 데이터의 실제 가치를 결정합니다.

  • 에셋 계층: 합성 데이터는 장면 에셋의 다양성에 극도로 민감합니다. 엔지니어링에서는 일반적으로 스캔 재구성(실제 물체 3D 스캔), 절차적 생성(매개변수화된 물체 및 레이아웃) 및 생성형 모델(텍스트-3D 에셋)의 세 가지 소스를 결합하여 에셋 라이브러리를 구축합니다. GR00T 합성 데이터 파이프라인은 Isaac Sim의 고충실도 렌더링 및 에셋 시스템을 기반으로 합니다.
  • 품질 검사 계층: 시뮬레이션 롤아웃이 본질적으로 올바른 것은 아닙니다. 스크립트 정책은 물리적으로 불가능한 궤적을 생성할 수 있으며, 물리 엔진의 수치적 이상 현상(관통, 폭발)도 데이터를 오염시킬 수 있습니다. 합성 데이터도 성공률 필터링, 동역학적 타당성 검사 및 시각적 합리성 샘플링이 필요합니다. 품질 검사 로직은 실제 데이터와 동일하지만, 시뮬레이션 실제 값을 활용하여 고도로 자동화할 수 있다는 점이 다릅니다.

21.3.2 데모 증강 및 자동 작업 생성

시뮬레이션 데이터의 핵심 문제는 "생성할 수 있는지"가 아니라 "무엇을 생성할지"입니다. 작업 분포의 다양성이 데이터의 일반화 가치를 결정합니다. 두 가지 대표적인 방법이 상호 보완적인 관점에서 이 문제를 해결합니다.

  • MimicGen: 데모 증강 프레임워크로, 소량의 인간 시드 데모를 템플릿으로 사용하여 시뮬레이션에서 물체 자세와 초기 조건을 교란함으로써 수십 개의 시드 데모를 수천 개의 변형 데모로 자동 확장합니다. 그 통찰력은 인간 데모의 가치가 "작업 전략의 분해 및 키프레임"에 있으며, 구체적인 기하학적 변형은 시뮬레이션을 통해 저렴하게 보완할 수 있다는 것입니다.
  • RoboGen: 대규모 언어 모델(LLM)로 구동되는 자동 작업 생성 프레임워크로, LLM이 다양한 작업을 제안하고, 해당 시뮬레이션 장면 코드와 보상 함수를 생성한 다음, 시뮬레이션이 자동으로 실행하고 데이터를 수집하여 작업 의미론적 수준에서 합성 데이터의 다양성을 확장합니다.

두 방법은 각각 데이터 엔진의 두 가지 증폭 차원에 해당합니다. 동일 작업 내 인스턴스 다양성과 작업 간 의미론적 다양성입니다. 도메인 무작위화(Domain Randomization) 방법(시뮬레이션에서 마찰 계수, 질량, 조명, 카메라 내/외부 파라미터 등 물리적 및 관측 파라미터를 무작위화)과 결합하면 훈련된 정책은 실제 세계의 분포 변화에 대해 더 강력한 견고성을 갖게 됩니다. 도메인 적응(Domain Adaptation) 방법은 소량의 실제 데이터를 사용하여 시뮬레이션에서 훈련된 정책의 분포를 정렬합니다.

flowchart LR
    A[소량의 인간 시드 데모<br/>또는 LLM 생성 작업 제안] --> B[시뮬레이션 작업 패밀리 구축<br/>MimicGen 교란 / RoboGen 생성]
    B --> C[도메인 무작위화 롤아웃<br/>물리적 및 관측 파라미터 샘플링]
    C --> D[자동화된 품질 검사<br/>성공률 및 타당성 필터링]
    D --> E[실제 데이터와 혼합 훈련]
    E --> F[실제 로봇 평가<br/>실패 모드 분석]
    F -->|작업 및 장면 지향적 보충| B

이 폐쇄 루프와 기존의 "일회성 라이브러리 구축" 접근 방식의 근본적인 차이점은 시뮬레이션 작업의 생성 방향이 실제 로봇 실패 모드에 의해 주도된다는 점입니다. 에셋 라이브러리에 현재 무엇이 있는지에 의해 주도되지 않습니다. 데이터 엔진의 각 반복은 정책의 능력 격차를 표적으로 보완합니다.

21.3.3 Sim-to-Real 격차 및 혼합 데이터 전략

시뮬레이션 데이터의 고유한 결함은 Sim-to-Real 격차(simulation-to-reality gap)입니다. 접촉 역학, 변형체, 센서 노이즈 및 액추에이터 동역학은 시뮬레이션에서 근사적으로만 모델링할 수 있으며, 시뮬레이션에서 우수한 성능을 보이는 정책이 실제 로봇에서 사용 가능하다는 것을 의미하지는 않습니다. 휴머노이드 로봇의 격차는 특히 두드러집니다. 작업이 양족 접촉 전환, 소프트 접촉 및 전신 동적 균형을 포함하기 때문이며, 이는 물리 엔진이 가장 정확하게 시뮬레이션하기 어려운 부분입니다.

엔지니어링 대응은 혼합 데이터 전략(co-training)입니다. 대규모 시뮬레이션 데이터는 작업 다양성과 집중적인 감독을 제공하고, 소량의 실제 데이터는 분포 변화를 보정합니다. 업계 관행에 따르면 VLA 유형 모델 훈련에서 실제 데이터와 합성 데이터의 비율, 그리고 합성 데이터의 도메인 무작위화 강도는 최종 실제 로봇 성공률에 모델 구조 자체보다 더 큰 영향을 미치는 경우가 많습니다. 따라서 데이터 엔진의 설계 목표는 데이터 양을 최대화하는 것이 아니라, 단위 훈련 컴퓨팅 성능당 실제 로봇 성능 향상을 최대화하는 것입니다. 이것이 모든 데이터 소스를 평가하는 최종 기준입니다.

Sim-to-Real 격차의 주요 원인과 일반적인 완화 방법은 다음과 같이 요약할 수 있습니다.

격차 원인 일반적인 표현 일반적인 완화 방법
접촉 역학 오류 소프트 접촉, 마찰 및 충돌 응답이 비현실적 접촉 파라미터 식별, 도메인 무작위화, 실제 로봇 접촉 데이터 미세 조정
액추에이터 동역학 오류 토크 응답 지연, 백래시 및 포화 미모델링 액추에이터 모델 식별(시스템 식별), 토크-전류 보정
관측 오류 렌더링 텍스처/조명이 실제 카메라와 다름 고충실도 렌더링, 실제 배경 텍스처 매핑, 카메라 노이즈 모델링
동역학 파라미터 오류 질량, 무게 중심, 관성이 실제 로봇과 일치하지 않음 파라미터 무작위화 및 온라인 적응, 실제 로봇 시스템 식별
분포 커버리지 부족 시뮬레이션 작업 분포가 실제 작업과 불일치 MimicGen/RoboGen 방식 작업 확장, 실제 롱테일 데이터 추가 수집

용어 설명: 데이터 엔진, Sim-to-Real, 도메인 무작위화

  • 데이터 엔진(data engine): "모델 결함 발견 → 데이터 지향적 생산 → 재훈련 → 재평가"의 폐쇄 루프를 중심으로 구축된 지속적인 데이터 생산 시스템으로, 일회성 데이터셋 구축 프로젝트가 아닙니다.
  • Sim-to-Real 격차: 물리적 모델링 오류와 관측 모델링 오류로 인한 시뮬레이션 분포와 실제 세계 분포 간의 체계적인 편향입니다.
  • 도메인 무작위화(domain randomization): 시뮬레이션에서 물리적 및 관측 파라미터에 무작위 교란을 적용하여 정책이 파라미터 변화에 둔감해지도록 학습시켜 Sim-to-Real 격차를 극복하는 방법입니다.

21.4 대표적인 공개 데이터셋과 벤치마크

공개 데이터셋은 업계 데이터 관행을 이해하기 위한 좌표계와 같습니다. 이는 사실상의 표준 데이터 규모, 모달리티 조합 및 작업 분류법을 정의하며, 중소 팀에게는 시작을 위한 훈련 재료를 제공합니다. 본 절에서는 "교차 본체 집계 – 단일 플랫폼 심층 수집 – 인간 중심"의 세 가지 유형으로 현재 가장 대표적인 자원을 정리하고, 마지막으로 엔지니어링 선택 기준을 제시합니다.

21.4.1 교차 본체 집계 데이터셋

단일 기관의 데이터 수집 능력에는 한계가 있으므로, 기관 간 집계는 데이터 규모를 빠르게 확장하는 중요한 경로가 됩니다.

Open X-Embodiment 데이터셋은 이 경로의 이정표입니다. 대규모 집계 로봇 학습 데이터셋으로, 20여 개 기관과 20여 종의 실제 로봇 플랫폼에서 수집된 시연 데이터를 통합하여 표준화된 데이터 형식과 작업 설명으로 정리했으며, 교차 체현 VLA 사전 훈련에 널리 사용됩니다. 함께 제공되는 RT-X 시리즈 모델은 "교차 본체 양성 전이" 가설을 검증했습니다. 즉, 다른 로봇 데이터에서 공동 훈련하면 대상 로봇이 보지 못한 작업에서 성능을 향상시킬 수 있습니다. 휴머노이드 로봇의 경우, Open X-Embodiment는 방대한 "범용 조작 사전 지식"을 제공하며, 휴머노이드 전용 데이터는 이를 기반으로 미세 조정만 하면 된다는 점에서 의미가 있습니다.

ARIO 전 로봇 통합 데이터셋은 통합 데이터 표준을 제시하며, 실제, 시뮬레이션 및 형식 변환 출처의 약 300만 개 에피소드를 수집하여 다양한 로봇 형태를 포괄합니다. 그 가치는 표준화된 본체 설명과 통합 데이터 스키마에 있으며, 이질적인 출처의 데이터를 동일한 훈련 파이프라인에서 혼합하여 사용할 수 있게 합니다.

용어 설명: 교차 본체 양성 전이, 에피소드, 데이터셋과 벤치마크

  • 교차 본체 양성 전이(positive cross-embodiment transfer): 다른 로봇 플랫폼의 데이터에서 공동 훈련한 후, 대상 로봇이 자체 작업에서 성능이 오히려 향상되는 현상. Open X-Embodiment의 RT-X 실험에서 처음으로 대규모로 이 효과를 검증했습니다.
  • 에피소드(episode/궤적): 한 번의 전체 작업 실행 기록 단위로, 일반적으로 작업 시작부터 성공/실패 판정 기준이 트리거될 때까지의 기간 동안 모든 모달리티의 동기화된 시퀀스를 포함합니다.
  • 데이터셋과 벤치마크(dataset vs. benchmark): 데이터셋은 훈련 재료를 제공합니다. 벤치마크는 추가로 작업 분할, 평가 프로토콜 및 지표를 규정하여 다양한 방법을 공정하게 비교하는 데 사용됩니다. 동일한 자원이 종종 두 가지 역할을 동시에 수행합니다.

집계 경로와 상호 보완적인 것은 단일 기관이 통일된 플랫폼에서 심층 수집하는 방식입니다. 이러한 데이터셋은 본체 일관성과 작업 체계성에서 더 큰 장점을 가집니다:

데이터셋 규모 및 구성 특징 및 용도

AgiBot World를 예로 들면, 해당 논문 AgiBot World Colosseo는 "데이터 팩토리" 방식의 수집 조직 방법을 보여줍니다. 통일된 본체, 통일된 수집 프로토콜, 가정, 산업, 상업 등 여러 장면을 포괄하며, 언어 명령어, 다중 시점 관찰, 본체 상태 및 관절 시퀀스를 통일된 다중 모달리티 표현으로 인코딩하여 전신 VLA 모델 훈련에 직접 사용됩니다. 이러한 "자체 구축 본체 + 자체 구축 데이터 + 자체 훈련 모델"의 수직 통합 모델은 선두 휴머노이드 로봇 회사의 표준 구성이 되고 있습니다.

DROID는 또 다른 조직 모델인 분산 연합 수집을 대표합니다. 여러 연구실이 통일된 데이터 스키마와 하드웨어 참조 설계에 따라 각자의 실제 환경에서 병렬로 수집한 후, 최종적으로 라이브러리로 통합합니다. 이 모델은 본체와 프로토콜의 높은 일관성을 희생하는 대신, 단일 기관으로는 도달할 수 없는 환경 다양성을 얻습니다. 서로 다른 연구실의 조명, 배경, 물체 및 조작 스타일 차이는 그 자체로 시각 및 환경 견고성을 향상시키는 자연스러운 데이터 증강입니다. 두 모델의 선택은 본질적으로 "일관성"과 "다양성" 간의 균형입니다. 미세 조정 단계에서는 전자를 선호하고, 사전 훈련 단계에서는 후자를 선호합니다.

RH20T와 OmniAction은 각각 모달리티 차원에서 모범을 보였습니다. 전자는 접촉이 풍부한 작업(삽입/분리, 조립, 닦기)에서 힘 감각과 오디오의 대체 불가능성을 입증했습니다. 순수 시각 전략은 이러한 작업에서 종종 가림으로 인해 실패합니다. 후자는 음성 명령어와 환경 소리를 컨텍스트에 통합하여 "로봇이 능동적으로 상호 작용을 시작"하는 데 필요한 다중 모달리티 신호 기반을 탐구했습니다. 휴머노이드 로봇의 경우, 이러한 모달리티 탐구의 의미는 휴머노이드 플랫폼의 궁극적인 시나리오가 인간과 로봇이 공존하는 개방된 환경이며, 단일 모달리티 데이터로는 이 시나리오에 필요한 이해 능력을 지원할 수 없다는 점에 있습니다.

21.4.3 인간 중심 및 전신 운동 데이터

로봇 본체 데이터 외에도 세 가지 유형의 "인간 측" 데이터셋은 휴머노이드 로봇 훈련 파이프라인에서 대체 불가능한 역할을 합니다. 이들의 공통된 가치는 인간 데이터의 규모는 수백만 시간 단위인 반면, 로봇 데이터의 규모는 수백만 개 단위라는 점에 있습니다. 인간 데이터를 사용하여 "작업을 어떻게 수행해야 하는지"에 대한 의미론 및 운동 사전 지식을 학습하고, 로봇 데이터를 사용하여 "로봇이 구체적으로 어떻게 수행하는지"에 대한 체현 구현을 해결하는 것은 데이터 규모의 격차를 극복하는 현실적인 경로입니다. 대표적인 네 가지 자원은 다음과 같습니다:

  • AMASS 동작 데이터셋: 대규모의 깨끗한 인체 모션 캡처 데이터로, 매개변수화된 인체 모델로 통합되어 전신 운동 전략 시뮬레이션 사전 훈련의 사실상 표준 출처입니다.
  • Ego4D 1인칭 시점 비디오 데이터셋: 3000시간 이상의 1인칭 시점 비디오로, 1인칭 기술 및 손-물체 상호 작용 이해 학습에 사용됩니다.
  • Epic-Kitchens 데이터셋: 1인칭 시점 주방 상호 작용 비디오로, 조작 시퀀스 및 활동 이해 연구에 널리 사용됩니다.
  • HumanPlus Shadowing 데이터셋: 휴머노이드 로봇이 그림자처럼 인간의 움직임을 따라가며 수집한 데이터로, 인간 시연과 로봇 전신 상태를 연결하여 전신 모방 및 원격 조작 전략 훈련에 사용됩니다.

21.4.4 데이터셋 선택을 위한 엔지니어링 기준

많은 데이터셋에 직면했을 때, 엔지니어링 선택은 다음 기준을 따를 수 있습니다:

  1. 본체 일치 우선: 대상 로봇 본체(또는 이와 유사한)의 데이터 전이 비용이 가장 낮습니다. 교차 본체 데이터는 미세 조정보다 사전 훈련에 적합합니다.
  2. 모달리티 완전성: 훈련에 필요한 모든 모달리티(예: 접촉 작업에 대한 힘 감각, VLA에 대한 언어)가 포함되어 있는지 확인합니다.
  3. 작업 및 장면 범위: 단순히 개수만 추구하기보다는 대상 애플리케이션 분포를 체계적으로 포괄하는 데이터셋을 우선 선택합니다.
  4. 프로토콜 및 라이선스: 데이터 라이선스가 상업적 사용을 허용하는지, 수집 프로토콜(캘리브레이션, 좌표계, 시간 동기화 방식)이 재현 가능한지 확인합니다.
  5. 혼합 가능성: 데이터 형식을 통합 스키마(예: LeRobot 형식 또는 RLDS)로 쉽게 변환할 수 있는지, 자체 데이터와 혼합 훈련이 가능한지 확인합니다.

참고로, 공개 데이터셋의 실제 가치는 종종 "직접 배포 가능한 전략을 훈련하는 것"이 아니라, 사전 훈련 기반으로서 자체 데이터의 콜드 스타트 비용을 낮추고, 다양한 방법을 공정하게 비교하기 위한 공공 벤치마크 역할을 하며, 데이터 프로토콜 설계의 참조 프레임워크를 제공하는 데 있습니다. 공개 데이터를 능력의 상한선을 넘는 보충 자료가 아닌 대체재로 보는 것이 데이터 자산에 대한 보다 현실적인 관점입니다.

21.5 데이터 엔지니어링: 형식, 저장, 비용 및 규정 준수

데이터가 수집 단계를 벗어나면 순수한 소프트웨어 엔지니어링 영역으로 들어갑니다. 형식은 상호 운용성을 결정하고, 품질 검사는 유효 데이터 비율을 결정하며, 저장 및 대역폭은 운영 비용을 결정하고, 규정 준수는 비즈니스 경계를 결정합니다. 이 절에서는 이 네 가지 측면에 대한 엔지니어링 실무 요점과 정량적 비용 추정 예시를 제시합니다.

21.5.1 데이터 형식 및 도구 체인

다중 모드, 다중 샘플링 속도의 로봇 데이터는 저장 형식에 특별한 요구 사항을 제기합니다. 비디오를 효율적으로 압축하는 동시에 각 모드의 시간 동기화와 무작위 접근 성능을 보장해야 합니다. 현재 주류 오픈 소스 도구 체인은 LeRobot입니다. 데이터 세트 형식, 원격 조작, 훈련 및 배포를 포괄하는 엔드투엔드 PyTorch 라이브러리입니다. 그 데이터 형식은 표준화된 스키마로 다중 시점 비디오, 상태 및 동작 시퀀스를 구성하며, 스트리밍 읽기와 증분 추가를 지원하여 학계와 소규모 팀이 데이터 파이프라인을 구축하는 사실상의 출발점이 되었습니다. Open X-Embodiment에서 채택한 RLDS(Reinforcement Learning Datasets) 형식은 널리 따르는 또 다른 표준으로, 에피소드 수준의 메타데이터와 교차 데이터 세트 상호 운용성을 강조합니다.

어떤 형식을 사용하든 데이터 스키마 설계에는 다음 필드가 명시적으로 포함되어야 합니다. 본체 식별자 및 캘리브레이션 파라미터, 각 모드의 타임스탬프 및 좌표계 정의, 작업 설명 및 성공 레이블, 수집 프로토콜 버전. 이러한 메타데이터가 없는 "베어 트레이젝토리"는 이후 혼합 훈련 및 추적에서 높은 비용을 초래합니다.

세 가지 일반적인 형식 솔루션의 엔지니어링 장단점은 다음과 같습니다.

솔루션 대표 장점 한계

실무에서 흔한 아키텍처는 "수집 측에서는 자체 구축 형식으로 효율성을 보장하고, 데이터 레이크에 적재할 때는 LeRobot/RLDS 등 표준 형식으로 변환하여 훈련에 사용"하는 것입니다. 두 형식 간에는 버전 관리되는 변환기 세트가 유지됩니다.

21.5.2 데이터 품질: 동기화, 필터링 및 중복 제거

데이터 품질 엔지니어링의 핵심 지표는 다음과 같습니다.

  • 시간 동기화 오차: 다중 카메라, 본체 상태 및 힘 신호의 타임스탬프 정렬 정밀도. 일반적으로 밀리초 수준 이내여야 하며, 그렇지 않으면 고속 작업에서 "관측-행동" 쌍에 체계적인 오정렬이 발생합니다.
  • 트레이젝토리 수준 품질 검사: 작업 성공률, 비정상 중단, 작업자 오류 구간의 자동 또는 반자동 제거.
  • 분포 중복 제거 및 재가중치 부여: 고빈도 반복 시나리오가 훈련 분포를 지배하는 것을 방지합니다. 롱테일 시나리오에 대한 오버샘플링은 강건성을 향상시키는 일반적인 방법입니다.
  • 의미론적 주석: 작업 명령, 세분화된 하위 작업, 객체 및 장면 레이블은 VLA 훈련 및 검색 기반 데이터 혼합에 직접 사용될 수 있습니다.

강조할 가치가 있는 것은 유효 데이터 비율이라는 복합 지표입니다. 원시 수집량을 \(D_{raw}\)라고 하고, 프레임 드롭 제거, 동기화 검증, 성공률 필터링 및 중복 제거 후 사용 가능한 양을 \(D_{eff}\)라고 하면, 유효 데이터 비율 \(r = D_{eff}/D_{raw}\)입니다. 프로토콜화된 관리가 부족한 수집 프로젝트에서 \(r\)은 50% 미만으로 낮아질 수 있습니다. 이는 저장, 대역폭 및 주석 예산의 거의 절반이 무효 데이터에 소비된다는 것을 의미합니다. 품질 검사를 최대한 수집 측으로 앞당기는 것(실시간 경고, 현장 재수집)이 \(r\)을 높이는 가장 효과적인 방법입니다.

21.5.3 저장, 대역폭 및 비용: 정량적 추정

데이터 인프라 비용은 간단한 규모 모델로 추정할 수 있습니다. 단일 수집 로봇에 \(N_c\)대의 카메라가 있고, 각 카메라의 비트레이트가 \(r_c\)(압축 후)이며, 상태/힘 등 저대역폭 신호의 총 비트레이트가 \(r_s\)라고 가정하면, 단일 로봇의 시간당 원시 데이터 양은 다음과 같습니다.

\[ D_{hour} = (N_c \cdot r_c + r_s) \times 3600 \text{ s} \]

\(M\)개의 수집 워크스테이션이 있고, 하루에 \(H\)시간 유효 수집하는 데이터 팩토리의 일일 증분은 \(M \cdot H \cdot D_{hour}\)이며, 연간 저장 요구 사항과 업링크 대역폭 요구 사항이 결정됩니다. 다음 Python 예제는 일반적인 데이터 팩토리의 추정치를 제공합니다.

# 데이터 팩토리 저장 및 대역폭 추정
N_c = 4          # 카메라 수: 헤드 스테레오 + 양손목 + 3인칭 (대표값 4로 설정)
r_c = 8e6        # 카메라당 압축 후 비트레이트 (bit/s), 1080p30 H.265 일반값
r_s = 2e6        # 상태/힘/오디오 총 비트레이트 (bit/s), 타임스탬프 및 메타데이터 포함
M = 100          # 수집 워크스테이션 수
H = 6            # 워크스테이션당 일일 유효 수집 시간

D_hour = (N_c * r_c + r_s) * 3600 / 8          # 단일 로봇 시간당 바이트 수
D_day  = D_hour * M * H                         # 일일 증분 (바이트)
D_year = D_day * 300                            # 연간 유효 수집일 300일 기준

print(f"단일 로봇 시간당 데이터량: {D_hour/1e9:.1f} GB")
print(f"데이터 팩토리 일일 증분:   {D_day/1e12:.2f} TB")
print(f"연간 저장 요구량:       {D_year/1e15:.2f} PB")
print(f"필요 평균 업링크 대역폭: {D_day*8/(H*3600)/1e9:.1f} Gbit/s (수집 시간대 집중)")

일반적인 실행 결과는 다음과 같습니다. 100개 워크스테이션 규모의 데이터 팩토리 일일 증분은 10 TB 수준이며, 연간 저장 요구량은 PB 수준에 달합니다. 이에 해당하는 클라우드 저장 및 CDN 반송 비용은 연간 수백만 위안에 달할 수 있습니다. 여기에는 수집 하드웨어 감가상각, 작업자 인건비 및 주석 비용이 포함되지 않습니다. 따라서 엣지 측 사전 필터링(성공 구간 및 핵심 모드만 반송), 계층형 저장(핫 데이터 SSD, 콜드 데이터 객체 저장) 및 비디오 코딩 최적화가 비용을 통제하는 세 가지 주요 레버입니다.

21.5.4 훈련 측의 데이터 혼합 및 커리큘럼 설계

데이터가 훈련 파이프라인에 들어가기 전 마지막 엔지니어링 결정은 혼합(mixture)입니다. 각 데이터 소스, 작업 및 모드가 어떤 가중치로 훈련에 참여할지 결정하는 것입니다. 경험적으로 균형을 맞춰야 하는 모순은 다음과 같습니다.

  • 실제 vs. 합성: 합성 데이터는 다양성을 제공하고, 실제 데이터는 분포를 보정합니다. 합성 데이터 비율이 너무 높으면 실제 로봇 성능이 오히려 떨어집니다. 일반적으로 실제 로봇 평가 결과에 따라 혼합 비율을 반복적으로 조정해야 하며, 한 번에 설정하는 것이 아닙니다.
  • 교차 본체 vs. 본체 전용: Open X-Embodiment 스타일의 교차 본체 데이터는 일반적인 사전 지식을 제공하고, 본체 전용 데이터는 최종 정밀도를 결정합니다. 일반적인 방법은 먼저 교차 본체 사전 훈련 후 본체 미세 조정을 하거나, 비율에 따라 혼합 훈련하고 손실에서 본체 전용 데이터에 가중치를 부여하는 것입니다.
  • 작업 균형 vs. 롱테일 가중치 부여: 균일 샘플링은 고빈도 단순 작업이 그래디언트를 지배하게 만듭니다. 작업 난이도 및 실패율에 따른 역빈도 가중치 부여는 롱테일 능력을 향상시키는 일반적인 방법이지만, 과도한 가중치 부여는 훈련 불안정을 초래할 수 있습니다.

혼합 자체는 모델 하이퍼파라미터와 동등하게 중요한 "데이터 하이퍼파라미터"로 간주되어 실험 관리에 포함되어야 합니다. 각 훈련마다 정확한 혼합 비율, 데이터 버전 및 샘플링 시드를 기록하여 결과의 재현성과 원인 추적 가능성을 보장해야 합니다.

21.5.5 규정 준수, 개인정보 보호 및 보안

휴머노이드 로봇 데이터는 본질적으로 민감한 정보를 포함합니다. 1인칭 시점 비디오는 얼굴, 가정 내부 및 상업 시설을 촬영할 수 있으며, 차량 군집 반송 데이터는 고객 현장을 포함합니다. 엔지니어링 측면에서 필요한 사항은 다음과 같습니다.

  • 수집 측 비식별화: 얼굴/번호판 자동 블러 처리, 음성 키워드 필터링, 가능한 한 엣지 측에서 완료;
  • 정보에 기반한 동의 및 계약: 수집 현장의 인원 고지, 데이터 사용 목적 및 보존 기간 약정;
  • 접근 제어 및 감사: 계층별 권한, 다운로드 기록, 버전 추적 가능;
  • 국경 간 규정 준수: 각 관할 구역은 데이터 해외 반출에 대해 다른 요구 사항을 가지며, 차량 군집 글로벌 배포 시 지역별 저장이 필요합니다.

이러한 요구 사항은 데이터 아키텍처 설계에 역으로 영향을 미칩니다. 예를 들어 "엣지 측 비식별화 후 데이터 레이크 적재"는 파이프라인의 선택 사항이 아닌 필수 단계가 됩니다. 실행 가능한 데이터 규정 준수 체크리스트에는 일반적으로 수집 전(현장 승인, 인원 고지, 계약 서명), 수집 중(실시간 비식별화, 민감 장면 자동 녹화 중지), 데이터베이스 적재 전(2차 비식별화 검토, 민감 메타데이터 분리), 사용 중(최소 권한 접근, 훈련 목적 감사), 폐기 시(만기 삭제, 권한 철회 후 연쇄 정리)의 다섯 단계 제어 지점이 포함됩니다. 규정 준수 비용은 저장 비용과 마찬가지로 데이터 예산에 포함되어야 하며, 사후 패치가 되어서는 안 됩니다.

21.6 장 요약

이 장에서는 휴머노이드 로봇 데이터 인프라의 전체 수명 주기를 체계적으로 설명했습니다. 핵심 관점은 다음과 같이 요약할 수 있습니다:

  1. 데이터는 휴머노이드 로봇 시대의 전략적 생산 요소입니다. VLA 및 모방 학습 패러다임에서 모델의 성능 상한은 데이터의 규모, 다양성 및 품질에 의해 공동으로 결정됩니다. 데이터 플라이휠은 선도 기업의 가장 깊은 해자 중 하나입니다.
  2. 데이터 소스는 피라미드를 구성합니다. 인터넷 비디오, 인간 중심 비디오, 모션 캡처, 시뮬레이션 합성 및 실제 원격 조작 데이터는 비용과 충실도에 따라 계층화되며, 현대 훈련 파이프라인은 커리큘럼 기반 혼합을 통해 각 계층의 데이터를 사용합니다.
  3. 원격 조작과 시뮬레이션 합성은 두 가지 주요 생산 엔진입니다. ALOHA/Mobile ALOHA 방식의 마스터-슬레이브 원격 조작, UMI/HuDOR 방식의 로봇 없는 수집, OptiTrack/AMASS 방식의 모션 캡처는 실제 또는 준실제 데이터를 제공합니다. Isaac Sim/MuJoCo/Genesis에 MimicGen/RoboGen 방식의 데이터 엔진을 더해 확장 가능한 합성 데이터를 제공하며, 도메인 무작위화와 혼합 훈련을 통해 Sim-to-Real 격차를 해소합니다.
  4. 공개 데이터셋은 전체 분야를 가속화했습니다. Open X-Embodiment는 교차 본체 사전 훈련을 검증했으며, AgiBot World는 휴머노이드 로봇 데이터 팩토리의 실현 가능성을 보여주었습니다. DROID, BridgeData V2, RH20T, OmniAction, ARIO 등은 각각 초점이 다르며, 선택 시 본체 일치성, 모달리티 완전성 및 작업 범위를 기준으로 삼아야 합니다.
  5. 데이터 엔지니어링이 단위 비용을 결정합니다. LeRobot과 같은 툴체인은 형식과 프로토콜을 통일합니다. 시간 동기화, 품질 검사 및 중복 제거는 유효 데이터 비율을 결정합니다. 저장, 대역폭 및 규정 준수는 무시할 수 없는 운영 비용을 구성하므로 아키텍처 설계 초기에 정량적으로 계획해야 합니다.
  6. 데이터 인프라의 최종 지표는 실제 로봇 성능입니다. 처리량, 동기화 정밀도, 유효 데이터 비율, 단위 비용은 모두 과정 지표일 뿐입니다. 모든 데이터 소스 및 비율 결정은 궁극적으로 통제된 실제 로봇 평가를 통해 판단되어야 합니다.

향후 몇 년간 세 가지 트렌드를 지속적으로 주목할 가치가 있습니다. 첫째, 데이터 소스로서의 월드 모델 — 학습된 월드 모델을 물리 엔진 대신 사용하여 "상상 궤적"을 생성하면 합성 데이터 비용을 한 자릿수 더 낮출 수 있지만, 물리적 현실성은 여전히 실제 데이터로 고정해야 합니다. 둘째, 데이터-모델 공동 스케일링 법칙의 실증적 확립, 즉 "어떤 비율에서, 얼마나 많은 어떤 데이터를 추가하면 얼마나 많은 실제 로봇 성능을 얻을 수 있는지"에 대한 답변을 통해 데이터 투자가 경험 기반에서 정량적 의사 결정으로 전환됩니다. 셋째, 기업 간 데이터 자산화 및 거래의 싹, 데이터 형식과 규정 준수 표준의 통일이 그 전제 조건입니다. 어떤 경로를 따르든 데이터 인프라는 휴머노이드 로봇 엔지니어링 투자의 가장 큰 부분 중 하나를 오랫동안 차지할 것입니다.

데이터 인프라 위에는 이러한 데이터와 모델을 실시간 로봇 동작으로 변환하는 소프트웨어 런타임 환경이 있습니다. 제22장에서는 소프트웨어 미들웨어로 전환하여 ROS 2, DDS, 실시간 버스 및 계획 제어 구성 요소가 휴머노이드 로봇의 각 컴퓨팅 유닛을 결정론적 전체로 구성하는 방법을 논의합니다. 두 장을 함께 읽으면 완전한 체인을 얻을 수 있습니다: 데이터는 파이프라인에서 생산 및 소비되며, 파이프라인 자체는 미들웨어 위에서 실행됩니다.
