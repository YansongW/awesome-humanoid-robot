데이터 원칙

  • 출처 추적: 모든 개체·관계는 출처 URL과 접근 일자를 포함하며, 마이닝된 관계는 근거 문장과 규칙 ID를 기록합니다.
  • 검증 투명성: 카드는 verified / partially_verified / unverified 상태와 confidence를 표시하고, AI 생성 콘텐츠는 항상 표기합니다.
  • 추가 전용: 사실 오류를 제외하고 기존 콘텐츠를 덮어쓰지 않습니다.
  • 지속 감사: CI가 배포 전 품질 감사(댕글링·중복·고립 개체·완전성)를 실행하며, 고립 개체 비율은 2% 미만입니다.

기술 스택

지식 그래프는 Markdown + YAML 프런트매터(research/, data/relationships/)로 저장되며 스키마는 data/schema/v1/로 관리됩니다. 정적 사이트는 website/builder/가 중/영/한 3개 언어로 빌드하고, 로컬 GUI(web/, FastAPI)는 검색·서브그래프 시각화·LLM Q&A를 제공합니다. 수집·마이닝 파이프라인은 scripts/에 있으며, 루트 Makefile(make help)로 주요 명령을 실행할 수 있습니다.

면책 조항

로드맵과 Wiki의 실습 안내는 공개 자료와 이론을 바탕으로 정리되었으며 실기 검증을 거치지 않았습니다. 전기·기계·리튬 배터리 작업 시 안전 규정을 직접 확인하고 현지 법규를 준수하세요. 이 콘텐츠는 엔지니어링 납품물이나 안전 인증 근거가 아닙니다.
