数据纪律

  • 来源可追溯：每个实体与关系都携带来源 URL 与访问日期；挖掘得到的关系记录证据句与规则编号，可批量回溯。
  • 校验状态透明：卡片标注 verified / partially_verified / unverified 与 confidence，AI 生成内容一律明确标示。
  • 只增不改：迭代以补充为主；除非存在事实性错误，不覆盖既有内容。
  • 持续审计：CI 每次部署前运行质量审计（悬空关系、重复、零度实体、完整性），当前零度实体率低于 2%。

技术栈

知识图谱以 Markdown + YAML Frontmatter 存储（research/、data/relationships/），schema 受控（data/schema/v1/）；静态站由 website/builder/ 构建，支持中/英/韩三语；本地 GUI 由 web/（FastAPI）提供检索、子图可视化与基于图谱的 LLM 问答；摄取与挖掘流水线见 scripts/（AI4Sci 工作流 + 关系挖掘）。仓库根目录 Makefile 提供全部常用命令（make help）。

免责声明

Start the roadmap →

Data Discipline

  • Traceable sources: every entity and relationship carries source URLs and access dates; mined relationships record evidence sentences and rule ids for bulk review.
  • Transparent verification: cards show verified / partially_verified / unverified status and confidence; AI-generated content is always labeled.
  • Additive only: iterations extend content; existing material is never overwritten except for factual errors.
  • Continuous audit: CI runs quality audits before every deploy (dangling relations, duplicates, zero-degree entities, completeness); the zero-degree rate is now below 2%.

Tech Stack

The knowledge graph is stored as Markdown + YAML frontmatter (research/, data/relationships/) with a controlled schema (data/schema/v1/). The static site is built by website/builder/ in Chinese, English and Korean. A local GUI (web/, FastAPI) offers search, subgraph visualization and LLM Q&A over the graph. Ingestion and relationship-mining pipelines live in scripts/. Common commands are available via the root Makefile (make help).

Disclaimer

When working with electronics, machinery or lithium batteries, verify safety practices yourself and follow local regulations. Nothing here constitutes an engineering deliverable or safety certification.
