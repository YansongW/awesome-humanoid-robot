# 选型手册：传感器（Sensor）怎么选

> ⚠️ 本路线图内容基于公开资料与理论知识整理，未经实机验证；涉及电气与机械操作时请自行核实安全规范。

传感器选型的常见错误是"按整机厂宣传片配齐"：RealSense 上头了、六维力买了、触觉阵列也想装，结果数据根本来不及处理，或者买回来发现控制频率被 USB 带宽拖死。本手册的思路反过来：先把你的机器人要完成的任务拆成"必须知道什么物理量"，再为每类物理量选刚好够用的传感器。原理背景见 [第 5 章 传感与感知硬件](/wiki/chapters/chapter-05/)，供应商格局见 [第 7 章 供应商地图](/wiki/chapters/chapter-07/)。

## 一、感知需求分解：你的机器人必须"知道"什么

把传感器分成三层，逐层回答"我真的需要吗"：

### 第 1 层：本体感觉（Proprioception）——关节角度与躯干姿态

【做什么】列出每个主动关节的位置反馈方案（电机端编码器、输出端编码器或双编码器），以及躯干/足部的 IMU 配置（装几颗、采样率多少）。

【为什么】本体感觉是控制闭环的命脉，缺了它机器人连"自己的腿在哪"都不知道。[关节编码器 Joint Encoder](/entry/ent_component_joint_encoder_2024/)为电机控制提供高分辨率位置反馈；值得注意的是"装在哪一端"：电机端编码器看不到减速器背隙与柔性变形，做精密力控的平台会加输出端编码器构成双编码器——Open Dynamic Robot Initiative 的 BLMC 执行器就是"双编码器（电机端/输出端）"设计（来源：调研档案 open-dynamic-robot-initiative.md）。低预算方案也有活路：Berkeley Humanoid Lite 用 $3 的 AS5600 磁编码器（来源：调研档案 berkeley-humanoid-lite.md），躯干姿态用一颗手机级 BNO085 IMU 也完成了零样本 sim-to-real 行走（同档案）。[IMU](/entry/ent_component_imu_2024/)负责姿态估计，与编码器、视觉融合提供高频惯性参考。

【你的情况怎么分析】位置控舵机方案（如 Dynamixel）编码器内置，这一层零成本解决；自研 QDD 关节至少要电机端编码器，要不要输出端编码器取决于你是否做力控（做→要）。IMU 起步一颗躯干级即可；做动态行走研究的建议足部加 IMU 或触地传感，BRUCE 双足机把 IMU 采样做到 2 kHz 就是这个目的（来源：调研档案 bruce-westwood.md）。

### 第 2 层：力觉（Force Sensing）——末端力、踝力、指尖触觉

【做什么】决定三件事：腕部/踝部要不要[六维力传感器 Six-axis F/T Sensor](/entry/ent_component_six_axis_force_torque_sensor_2024/)（装在哪个法兰、量程多少）；灵巧手指尖要不要[触觉传感器阵列 Tactile Sensor Array](/entry/ent_component_tactile_sensor_array_2024/)；还是先用执行器电流环估力顶上。

【为什么】六维力传感器同时测三向力与三向力矩，是力控装配、拖动示教、双足踝关节 ZMP 估计的硬通货——THORMANG3 在双脚踝各装一颗 ATI Mini58（来源：调研档案 thormang3.md）。但它是全机最贵的传感器之一，且量程与精度互相打架（量程买大了分辨率就浪费）。触觉阵列测接触压力/剪切/纹理，支撑灵巧操作的滑移检测；预算紧张时可用 FSR 阵列入门——Poppy 人形机 2014 年配置就是 16 个 FSR 力传感器（来源：调研档案 poppy-humanoid.md）。

【你的情况怎么分析】只做行走不做操作 → 电流环估力+足端开关式接触传感即可，省下六维力的钱；要做装配/打磨类操作研究 → 腕部六维力值得投资，量程按"最大接触力×2"选；做灵巧手抓取研究 → 触觉阵列优先于腕部六维力，因为抓取滑移主要发生在指尖。

### 第 3 层：外感知（Exteroception）——看外面的世界

【做什么】明确外感知任务的类型：近距离操作场景理解（0.3–3 m）、室内导航建图（3–20 m）、还是户外/大空间。三者对应完全不同的传感器。

【为什么】[RGB-D 相机](/entry/ent_component_rgbd_camera/)三条深度路线——结构光、ToF、双目立体——各有适用距离与光照约束：结构光精度高但怕强光、双目被动但依赖纹理。头部视觉主流是 [Intel RealSense](/entry/ent_component_intel_realsense_depth_camera_2024/)（D435i/D455 被青龙、G1 等平台采用，来源：openloong-qinglong 调研档案、宇树卡片）；导航建图主力是激光雷达，[Livox Mid-360](/entry/ent_component_livox_mid_360_lidar_2024/) 360° 环视、265 g，是四足/人形 SLAM 的热门选择（来源：Livox 卡片）。

【你的情况怎么分析】只做桌面抓取 → 一颗 RGB-D 够了，激光雷达是浪费；要做室内行走导航 → RGB-D（近距避障）+ 360° LiDAR（建图）是成熟组合；纯视觉研究（VLA、视觉伺服）→ 把预算花在相机帧率与全局快门上，而不是堆 LiDAR。

## 二、分品类选型表：关键指标 → 怎么影响你的机器人 → 价位档

### 编码器（Encoder）

| 关键指标 | 怎么影响你的机器人 | 价位档参考 |
|---|---|---|
| 分辨率（位数） | 决定位置控制精度与速度估算质量；17 bit 以上关节才能做平稳低速力控 | 磁编码器 AS5600（12 bit）约 $3（BHL 档案）；高分辨率光电/绝对式需询价 |
| 绝对式 vs 增量式 | 绝对式上电即知角度，人形机关节多，增量式每次上电回零很痛苦 | 绝对式贵，按需 |
| 安装端（电机端/输出端） | 输出端编码器让你"看见"减速器背隙与柔轮变形，是力控透明度前提 | 双编码器=两倍成本，力控平台值得（ODRI 档案） |

### IMU

| 关键指标 | 怎么影响你的机器人 | 价位档参考 |
|---|---|---|
| 陀螺偏置稳定性 | 决定姿态积分漂移速度；[IMU 卡片](/entry/ent_component_imu_2024/)示例 ADIS16475 为 2°/hr（运行中） | 手机级（BNO085，BHL 在用）几十元级；工业 MEMS（ADIS16475 级）价格未公开需询价；FOG 级另算 |
| 角随机游走（ARW） | 噪声直接进姿态估计，ADIS16475 为 0.15°/√hr | 同上 |
| 轴间对准误差 | 多传感器融合的安装对准工作量，ADIS16475 ±0.1° 出厂已校准 | 低端 IMU 需自己标定 |
| 采样率/接口 | 平衡控制环路需要高频姿态，SPI 比 I²C 延迟低；ADIS16475 为 SPI | BRUCE 做到 2 kHz（档案） |

### 六维力/力矩传感器

| 关键指标 | 怎么影响你的机器人 | 价位档参考 |
|---|---|---|
| 量程（F/M） | 按"最大接触力×2"选；神源生 MLL 典型 50/50/100 N、2/2/4 N·m，[OnRobot HEX-E QC](/entry/ent_component_ati_force_torque_sensor_2024/) ±100/±200 N、±10 N·m（来源：两卡片） | 国产模拟量型（MLL 级）需询价；进口 ATI Nano25（±250 N）级价格显著更高 |
| 精度/耦合误差 | 决定六轴解耦后力控的可信度；MLL 精度 ≤0.5% FS、耦合误差 ≤2% FS | 高精度型号更贵 |
| 过载能力 | 双足踝部必然经历冲击，MLL ≥300% FS、HEX-E 约 500% FS | 过载余量=寿命 |
| 采样率与接口 | 力控环要 ≥1 kHz；MLL 最高 1000 Hz（配采集器），HEX-E 走 EtherNet/IP 等工业总线 | 采集器/总线网关另计 |

### 触觉阵列

| 关键指标 | 怎么影响你的机器人 | 价位档参考 |
|---|---|---|
| 空间分辨率（taxel 密度） | 决定能否分辨接触形状与纹理，支撑滑移预判 | FSR 离散点最便宜（Poppy 用 16 点）；阵列式需询价 |
| 量测维度 | 只测法向压力 vs 三向力（含剪切）——剪切是滑移前兆 | 三向阵列贵得多，抓取研究才需要 |
| 曲面贴附与引线 | 指尖曲面安装与走线是工程大头，常比传感器本身更费工 | 评估柔性 PCB 方案 |

### RGB-D 相机（RealSense vs ZED）

| 关键指标 | 怎么影响你的机器人 | 价位档参考 |
|---|---|---|
| 理想量程与精度 | 操作场景关注 0.3–1 m 段精度：[RealSense](/entry/ent_component_intel_realsense_depth_camera_2024/) D435i 为 0.1–10 m、<2% @2 m；D455 为 0.6–6 m、<2% @4 m（来源：RealSense 卡片，官方 datasheet） | D435i 约 USD 199、D455 约 USD 299（公开市场参考，卡片口径） |
| 快门类型 | 头部在运动，全局快门（D455）避免卷帘畸变 | 一般全局快门型号略贵 |
| 内置 IMU | 视觉-惯性融合（VIO）省一颗外置 IMU：D435i 内置 BMI055 | i 后缀型号才有 |
| 算力依赖 | [ZED 立体相机](/entry/ent_component_zed_stereo_camera_2024/) onboard 深度+空间 AI，但深度计算依赖主机 GPU 生态（来源：ZED 卡片）；价格需自行向供应商确认 | RealSense 生态（librealsense+ROS）零算力门槛，ZED 需配 NVIDIA 主机 |

### 激光雷达（LiDAR）

| 关键指标 | 怎么影响你的机器人 | 价位档参考 |
|---|---|---|
| FOV | [Livox Mid-360](/entry/ent_component_livox_mid_360_lidar_2024/) 水平 360°、竖直 -7°~52°，一颗顶躯干环视（来源：Livox 卡片） | Mid-360 约 CNY 3,999（官方商城，卡片口径） |
| 探测距离与精度 | 40 m @10% 反射率、≤2 cm @10 m——室内导航建图绰绰有余 | 同左 |
| 点频与帧率 | 200,000 点/秒、10 Hz，决定动态场景更新率 | 同左 |
| 重量/功耗 | 265 g、6.5 W，头部/躯干可接受；内置 ICM40609 IMU 可做 LIO | 同左 |

## 三、标定与时间同步：传感器买对只完成一半

【做什么】制定标定计划：相机内参（棋盘格）、相机-LiDAR 外参、相机-IMU 外参与时间偏移、关节零位，并约定统一时间源（PTP 或硬件触发）。

【为什么】多传感器融合的一切算法都假设"外参已知、时钟同步"。[联合标定 Joint-Camera-IMU Calibration](/entry/ent_method_calibration_joint_camera_imu/)卡片给出了标准流程：标定靶多姿态采集 → 相机检测角点 + LiDAR 提取平面/角点 → PnP 初值 + ICP/非线性优化 → 最小化重投影误差验证；无标定靶时可用场景边缘对齐或互信息法在线 refinement。时间偏移没标好，运动中的融合结果就是"重影"，VIO 发散、点云着色错位都源于此。集成与测试的系统方法见 [第 11 章 装配、集成与测试](/wiki/chapters/chapter-11/)。

【你的情况怎么分析】单 RGB-D 起步只需做相机内参；上 LiDAR 就必须做相机-LiDAR 外参（卡片方法可直接照做）；做 VIO/多 IMU 融合再补时间标定。验收指标就用重投影误差，别凭眼睛看"差不多对齐了"。

## 四、两套推荐配置

### 新手最小套装（目标：走得起来、看得见、摔不坏）

- 关节反馈：智能舵机内置编码器，或自研关节用 AS5600 级磁编码器（$3/颗，BHL 档案口径）。
- 姿态：手机级 IMU 一颗（BNO085 级），先跑通姿态解算。
- 外感知：RealSense D435i 一颗（约 USD 199，卡片口径）——深度+RGB+IMU 一体，生态最成熟。
- 力觉：先不上，用电流环估力；足端可用开关/FSR 判触地。
- 分析逻辑：这套把每一分钱花在"控制闭环必需"上，行走与抓取演示都够；等瓶颈明确出现（姿态漂移大、抓握滑移检测不到）再逐项升级。

### 研究级套装（目标：力控/操作/导航论文级数据）

- 关节反馈：双编码器（电机端+输出端），输出端用高分辨率绝对式。
- 姿态：工业 MEMS IMU（ADIS16475 级：ARW 0.15°/√hr、偏置 2°/hr、±0.1° 对准，卡片口径）+ 躯干/足部分布式布置，采样往 1–2 kHz 做。
- 力觉：腕部六维力（国产 MLL 级或 ATI/OnRobot 级）+ 双足踝部六维力（THORMANG3 配置先例），灵巧手加触觉阵列。
- 外感知：RealSense D455（全局快门，约 USD 299）或 ZED（有 GPU 主机时）+ Livox Mid-360（约 CNY 3,999）做 360° 建图。
- 标定：全套内参/外参/时间同步标定，重投影误差纳入验收。
- 分析逻辑：研究级的核心是"数据可信度"——传感器贵不是目的，可标定、可溯源、时钟同步才是。

## 验收标准

- 有一份"物理量→传感器→安装位置→接口→采样率"清单，每项都能回答"没有它会怎样"。
- 每颗传感器的关键指标（量程/精度/带宽）与任务需求一一对应，且可溯源到卡片或 datasheet；无公开价的已标"需自行向供应商确认"。
- 标定计划落地：相机内参完成，装了 LiDAR 的完成相机-LiDAR 外参并记录重投影误差。
- 全传感器时间戳统一到同一时钟源，融合节点无肉眼可见的时间错位。

## 常见坑与排查

| 症状 | 可能原因 | 排查动作 |
|---|---|---|
| 姿态解算慢漂、越走越歪 | IMU 偏置稳定性不够或未做温补标定 | 查偏置稳定性指标；静置估偏置，必要时升级工业级 IMU |
| 行走中 VIO/融合发散 | 相机-IMU 时间偏移未标定 | 做时间同步标定；检查时间戳来源是否统一 |
| 点云着色错位、重影 | 相机-LiDAR 外参过期（碰撞后） | 重做多姿态标定，看重投影误差是否收敛 |
| 六维力读数零点乱漂 | 温漂或未去皮；安装应力串入 | 上电预热后去皮；检查法兰安装预紧力 |
| 六维力量程总打满 | 量程按"典型力"选而非"冲击×2" | 核对踝部冲击峰值；换大量程或加过载保护策略 |
| 深度图阳光下大片空洞 | 结构光/主动 IR 怕强光（路线固有） | 查深度技术路线；户外换被动双目或 LiDAR |
| 触觉阵列数据噪声大 | 引线受弯折干扰、供电纹波 | 固定引线走线；示波器查供电，加滤波 |
